# 보고서 작성 기준

> 모든 보고서에 적용되는 형식, 품질, 스타일 기준

---

## 공통 작성 원칙

### 형식 규칙
- 모든 보고서는 **Markdown** 형식으로 작성
- 파일명: `{기업명}_{보고서유형}_{YYYY-MM-DD}.md`
- 상단에 **메타 정보** 블록 포함 (작성일, 작성자, 기밀등급, 버전)
- 본문은 **목차 → 요약 → 본문 → 부록** 순서

### 메타 정보 블록 표준

```
---
제목: [보고서 제목]
작성일: YYYY-MM-DD
작성자: [작성자/에이전트명]
기밀등급: [공개/내부용/대외비/기밀]
버전: v1.0
대상 기업: [기업명] (해당 시)
보고 기간: [기간] (해당 시)
---
```

### 품질 기준
1. **정량적 근거**: 수치 주장에는 반드시 데이터 출처 명시
2. **객관성**: 긍정/부정 양면을 균형 있게 서술
3. **불확실성 표기**: 확인 안 된 정보는 `[미확인]`, 추정치는 `[추정]` 표기
4. **출처 명시**: 외부 데이터 인용 시 출처와 시점 명시
5. **일관성**: 동일 지표는 동일한 용어와 단위 사용

### 문체 규칙
- 간결하고 명확한 문장 사용
- 능동태 우선
- 전문 용어 첫 사용 시 영문 병기 (예: 번레이트(Burn Rate))
- 약어 첫 사용 시 풀네임 표기 (예: TAM(Total Addressable Market))
- 불필요한 수식어 자제

## 숫자 표기 규칙

| 규모 | 표기 방식 | 예시 |
|------|----------|------|
| 만원 단위 | 한글 병기 | 500만원 |
| 억원 단위 | 한글 병기 | 15억원 |
| 조원 단위 | 한글 병기 | 1.2조원 |
| 달러 | $ 기호 | $1.5M, $2B |
| 퍼센트 | % 기호 | 15.3% |
| 배수 | x 표기 | 3.5x |

## 차트/그래프 가이드

Markdown 내에서 시각화가 필요한 경우:
- ASCII 기반 간이 차트 사용
- 데이터 테이블로 대체
- 별도 시각화 도구 활용 시 이미지 링크

2. 담당자가 팩트체크 및 수정
3. 기밀등급에 따라 승인 절차 진행
4. 최종본 저장 및 배포
