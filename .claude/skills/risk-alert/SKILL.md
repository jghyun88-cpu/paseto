# 리스크 점검 프로세스

대상: $ARGUMENTS (기본값: all - 전체 포트폴리오)

## 실행 절차

### 1. 대상 확정 및 데이터 수집

1. 인자를 분석하여 점검 대상을 확정합니다
   - `all`: `ai-agent/data/portfolio/` 하위 전체 기업
   - 기업명 지정: 해당 기업만 점검
2. 대상 기업별 최신 KPI 데이터를 수집합니다
   - `ai-agent/data/portfolio/{기업}/kpi_*.md` (가장 최근 파일)
3. 이전 리스크 리포트를 확인합니다
   - `ai-agent/reports/risk/` 에서 가장 최근 리포트 로드

### 2. risk-monitor 에이전트 실행

`risk-monitor` 에이전트를 호출하여 다음을 점검합니다:

**재무 리스크:**
- 런웨이 경보 (3개월 미만 = 긴급, 6개월 미만 = 주의)
- 번레이트 급증 (MoM +20% 이상)
- 매출 하락 추세 (3개월 연속 하락 = 긴급)
- 단위 경제학 악화 (LTV/CAC < 3x)

**성장 리스크:**
- 핵심 KPI 급락 (전월 대비 30% 이상)
- 성장 정체 (3개월 연속 5% 미만 성장)
- 고객 이탈 증가

**팀 리스크:**
- 핵심 인력 이탈 정보
- 채용 지연 여부

**시장/규제 리스크:**
- WebSearch로 관련 산업 뉴스/규제 변화 확인
- 경쟁사 위협적 동향 파악

**실행력 리스크:**
- 마일스톤 달성 여부
- 계획 대비 실행 괴리

### 3. 경보 분류 및 종합

에이전트 결과를 바탕으로 종합 리스크 등급을 산정합니다:

```
🔴 긴급 (Critical):
  - 즉각 대응 필요
  - 긴급 항목 1개 이상 OR 주의 항목 3개 이상

🟡 주의 (Warning):
  - 면밀한 모니터링 필요
  - 주의 항목 1-2개

🟢 양호 (Normal):
  - 정상 범위 내
  - 모든 항목 양호
```

### 4. 이전 경보와 비교

- 이전 리스크 리포트가 있으면 경보 변화를 추적합니다
  - 신규 경보: 이전에 없었던 경보 발생
  - 개선: 이전 경보가 해소됨
  - 악화: 이전 주의 → 긴급으로 격상
  - 지속: 동일 경보 유지

### 5. 대응 권고안 작성

각 경보 항목에 대해 구체적 대응 방안을 제시합니다:
- **긴급**: 즉시 연락/미팅 필요, 구체적 논의 안건 제시
- **주의**: 다음 월간 미팅에서 확인, 모니터링 포인트 제시
- **양호**: 특이사항 없음, 성과 격려 포인트 제시

### 6. 저장

- `ai-agent/reports/risk/{날짜}_alert.md` 에 저장합니다
- 전체 점검: `2026-02-25_alert.md`
- 개별 점검: `2026-02-25_{기업명}_alert.md`

### 7. 결과 요약 보고

사용자에게 다음을 보고합니다:

**긴급 사항이 있는 경우:**
- 🔴 긴급 대응 필요 기업 및 사유
- 권고 즉시 조치 사항

**전체 요약:**
- 리스크 대시보드 (기업별 종합 등급)
- 주의 사항 요약
- 이전 대비 변화 사항
- 저장 경로

## 주의사항

- **데이터 기반 판단**: 감이 아닌 수치 기준으로 경보 발령
- **오탐 최소화**: 일시적 변동(계절성, 일회성 이벤트)과 구조적 문제를 구분
- **컨텍스트 고려**: 기업 단계(시드 vs 시리즈A)에 따라 기준 유연 적용
- **기밀 유지**: 외부 검색 시 기업명 + 미공개 정보 노출 금지
- **액션 지향**: 경보만 나열하지 않고, 구체적 대응 권고 포함
