## 2. 섹션별 분석 포인트

### Problem (문제)
- 문제의 크기: 이 문제로 인한 비용/손실이 얼마나 되는가?
- 문제의 빈도: 얼마나 자주 발생하는가?
- 문제의 긴급성: 해결이 시급한가, 아니면 "있으면 좋은" 것인가?
- 문제의 보편성: 얼마나 많은 사람/기업이 겪는가?
- 현재 해결 방식: 고객이 현재 어떻게 대처하고 있는가?

### Solution (솔루션)
- 문제와의 연결: 정의된 문제를 직접 해결하는가?
- 차별화: 기존 대안보다 어떤 점이 10x 나은가?
- 기술적 실현: 현재 기술로 구현 가능한가?
- 확장성: 규모가 커져도 작동하는가?
- 방어 가능성: 쉽게 복제될 수 있는가?

### Market Size (시장 규모)
- TAM 산출 방법: Top-down? Bottom-up? 둘 다?
- 데이터 출처: 신뢰할 수 있는 출처인가? (보고서, 통계청 등)
- 시장 정의: 시장을 너무 넓게/좁게 정의하지 않았는가?
- 성장률: 과거 데이터에 기반한 추정인가?
- SOM 합리성: 초기 목표 시장 점유가 현실적인가?

### Business Model (비즈니스 모델)
- 수익원: 명확하고 반복적인가?
- 가격: 고객의 지불 의사(WTP)에 부합하는가?
- 마진: 규모 확대 시 마진이 개선되는 구조인가?
- LTV/CAC: 3x 이상이거나 개선 추세인가?
- Lock-in: 전환 비용이 존재하는가?

### Traction (트랙션)
- 지표의 진정성: 허영 지표(Vanity Metrics) 아닌 실질 지표인가?
- 성장 추세: 상승, 정체, 하락 중 어느 단계인가?
- PMF 징후: "40% 테스트" 등 PMF 지표가 있는가?
- 코호트 분석: 초기 사용자의 유지율이 양호한가?
- 유기적 성장: 돈을 태워서 만든 성장인가, 유기적인가?

### Team (팀)
- Founder-Market Fit: 왜 이 팀이 이 문제를 풀기에 적합한가?
- 핵심 역량: CEO(비전/사업), CTO(기술), CPO(제품) 존재?
- 보완성: 팀원 간 역량이 겹치지 않고 보완적인가?
- 헌신도: 풀타임 여부, 기존 직장 퇴사 여부
- 트랙레코드: 이전 성과가 현재 사업과 연관이 있는가?

### Competition (경쟁)
- 정직성: 경쟁사를 솔직하게 인정하는가? ("경쟁사 없음"은 레드플래그)
- 분류: 직접/간접/잠재 경쟁사를 구분하는가?
- 포지셔닝: 차별화 포인트가 명확하고 방어 가능한가?
- 대기업 위협: 대기업이 쉽게 진입할 수 있는 영역인가?

### Financials (재무)
- 현실성: 하키스틱 성장 곡선에 합리적 근거가 있는가?
- 가정: 핵심 가정이 명시되어 있는가?
- 번레이트: 현재 지출 수준이 단계에 적합한가?
- 런웨이: 충분한가? (최소 12-18개월 권장)
- 자금 사용: 투자금 사용 계획이 합리적인가?

### Ask (투자 요청)
- 금액: 다음 마일스톤 달성에 충분한가?
- 밸류에이션: 단계와 트랙션에 비해 적정한가?
- 용도: 명확하고 마일스톤과 연결되는가?
- 희석률: 창업팀의 동기부여에 문제없는 수준인가?

## 3. 투자 매력도 종합 판단 매트릭스

```
            팀 강함
               │
    B (추천)   │   A (강력 추천)
               │
시장 작음 ─────┼───── 시장 큼
               │
    D (비추천) │   C (시장은 좋으나
               │      팀 보강 필요)
            팀 약함
```

- A: 큰 시장 + 강한 팀 = 투자 적극 검토
- B: 작은 시장이나 뛰어난 팀 = 피봇 가능성 고려
- C: 큰 시장이나 팀 미흡 = 팀 보강 조건부 검토
- D: 작은 시장 + 약한 팀 = 투자 보류

| 중간 | △ | IR 내 데이터만 존재, 검증 미완 |
| 낮음 | ? | 데이터 부족, 추정에 의존 |
