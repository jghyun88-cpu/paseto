# LP 보고서 생성 프로세스

보고 기간: $ARGUMENTS (기본값: 가장 최근 분기)

## 실행 절차

### 1. 보고 기간 확정 및 데이터 수집

1. 인자로 전달된 보고 기간을 확정합니다
   - Q1 = 1-3월, Q2 = 4-6월, Q3 = 7-9월, Q4 = 10-12월
   - H1 = 1-6월, H2 = 7-12월, annual = 1-12월
   - 미지정 시 현재 날짜 기준 가장 최근 완료 분기
2. `ai-agent/data/fund/` 에서 펀드 운용 데이터를 로드합니다
3. `ai-agent/data/portfolio/` 에서 전체 포트폴리오 KPI 데이터를 수집합니다
4. `ai-agent/reports/` 에서 기간 내 생성된 분석/스크리닝 보고서를 확인합니다

### 2. 에이전트 실행

**순차 실행:**
1. `portfolio-reporter` 에이전트로 포트폴리오 전체 현황 요약을 생성합니다
   - 기업별 KPI 대시보드, 상태 분류, 하이라이트 추출
2. `lp-report-writer` 에이전트를 호출하여 LP 보고서 초안을 작성합니다
   - 포트폴리오 요약 결과 + 펀드 운용 데이터를 전달
   - `.claude/skills/lp-report/report-template.md` 형식 적용
   - `.claude/rules/reporting-standards.md` 품질 기준 준수

### 3. 검토 포인트 자동 확인

보고서 초안에 대해 다음을 자동 점검합니다:
- [ ] 수치 일관성: 보고서 내 수치와 원본 데이터 일치 여부
- [ ] 기밀등급 표기: 메타 정보에 "기밀" 등급 명시 여부
- [ ] 민감 정보: 미공개 파이프라인 기업명, 비공개 조건 등 누출 여부
- [ ] 필수 섹션: 템플릿의 모든 필수 섹션 작성 여부
- [ ] 톤 & 스타일: 전문적/절제된 톤 유지 여부

### 4. 저장

- 보고서를 `ai-agent/reports/lp/{기간}_{날짜}.md` 에 저장합니다
- 파일명 예: `Q4-2025_2026-01-15.md`, `H2-2025_2026-01-15.md`

### 5. 결과 요약 보고

사용자에게 다음을 보고합니다:
- 보고 기간 및 커버리지
- 펀드 핵심 성과 지표 (IRR, MOIC)
- 포트폴리오 하이라이트 Top 3
- 주의 필요 사항
- 저장 경로

## 보고서 유형별 가이드

| 유형 | 포함 섹션 | 분량 목표 |
|------|----------|----------|
| 분기 (Q) | 전체 섹션 | 8-12페이지 |
| 반기 (H) | 전체 + 시장심층분석 + 포트폴리오 상세 | 15-20페이지 |
| 연간 (annual) | 전체 + 전략리뷰 + 내년계획 + 벤치마크 | 25-35페이지 |

## 주의사항

- LP 보고서는 **기밀** 등급입니다
- 모든 수치는 원본 데이터와 대조 검증이 필요합니다
- 보고서 완성 후 반드시 **담당자 검토**가 필요함을 안내합니다
- 미확정 정보에는 `[추정]` 또는 `[잠정]` 표기를 합니다
