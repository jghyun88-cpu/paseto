# 지원서 스크리닝 실행

대상: $ARGUMENTS

## 실행 절차

### 1단계: 대상 지원서 확인

지원서 파일을 찾습니다:
- `$ARGUMENTS`가 파일 경로이면 해당 파일을 직접 읽습니다
- 기업명이면 `ai-agent/data/applications/` 하위에서 검색합니다
- 파일을 찾을 수 없으면 사용자에게 경로를 요청합니다

### 2단계: 평가 기준 로드

다음 참조 파일을 로드합니다:
- `.claude/rules/evaluation-rubric.md` - 100점 만점 평가 루브릭
- `ai-agent/data/reference/evaluation-criteria.md` - 단계별 기대 수준, 산업별 포인트

### 3단계: 스크리닝 에이전트 실행

`application-screener` 에이전트를 호출하여 평가를 수행합니다.

에이전트에게 전달할 컨텍스트:
- 지원서 전체 내용
- 평가 기준
- 해당 배치 정보 (있는 경우)

### 4단계: 결과 저장

보고서를 다음 경로에 저장합니다:
- `ai-agent/reports/screening/{기업명}_screening_{YYYY-MM-DD}.md`
- 기업명은 영문 소문자, 공백은 하이픈으로 변환

### 5단계: 결과 요약 보고

사용자에게 다음을 보고합니다:

```
## 스크리닝 결과 요약

| 항목 | 결과 |
|------|------|
| 기업명 | |
| 총점 | XX/100 |
| 판정 | 통과/조건부통과/보류/탈락 |

### 핵심 강점
1.
2.
3.

### 주요 우려사항
1.
2.
3.

📄 상세 보고서: ai-agent/reports/screening/{파일명}
```

## 일괄 스크리닝

여러 지원서를 한 번에 스크리닝하는 경우:
- `ai-agent/data/applications/{배치폴더}/` 내 모든 지원서를 대상으로
- 각 지원서를 순차적으로 스크리닝
- 완료 후 종합 비교표를 생성합니다:

```
## 배치 스크리닝 종합 결과

| 순위 | 기업명 | 총점 | 팀 | 시장 | 트랙션 | BM | 정성 | 판정 |
|------|--------|------|-----|------|--------|-----|------|------|
| 1 | | /100 | /15 | /15 | /15 | /15 | /40 | |
| 2 | | /100 | /15 | /15 | /15 | /15 | /40 | |
```
