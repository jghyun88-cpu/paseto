  현재 구현 상태 (DocumentsTab 컴포넌트 기반)                                                                                                                                                  

   핵심 설계 원칙

  ▎ "같은 기업 ID + 같은 데이터룸" — 한 기업의 모든 문서를 단일 데이터룸에서 관리리                                                                                                                                                                                             
  DocumentsTab 컴포넌트는 allowedCategories prop으로 단계별 허용 카테고리를 제한하는 설계입니다. 현재 2곳에서 사용 중:                                                                         

  ┌─────┬───────────────────────────────────────────┬────────────────────────────┬────────────────────────┬────────────────────────────────────┐
  │  #  │                화면 (경로)                │             탭             │     허용 카테고리      │                상태                │
  ├─────┼───────────────────────────────────────────┼────────────────────────────┼────────────────────────┼────────────────────────────────────┤
  │ 1   │ 소싱관리 > 딜 상세 (/sourcing/deals/[id]) │ 기본정보 / 문서 / 미팅     │ ir, other              │ 구현 완료 (스크린샷에 보이는 화면) │
  ├─────┼───────────────────────────────────────────┼────────────────────────────┼────────────────────────┼────────────────────────────────────┤
  │ 2   │ 보육관리 > 상세 (/incubation/[id])        │ 기본정보 / 성장과제 / 문서 │ mentoring, report, poc │ 구현 완료                          │
  └─────┴───────────────────────────────────────────┴────────────────────────────┴────────────────────────┴────────────────────────────────────┘

  ---
  미구현 화면 (계획은 있으나 DocumentsTab 미배치)

  마스터 스펙 §26의 DocumentCategory Enum 8개 카테고리 기준으로, 각 단계에서 필요한 문서는 다음과 같지만 아직 문서 탭이 없는 화면들입니다:

  ┌─────┬────────────────────────────────────────────────────┬─────────────────┬─────────────────────────────────────────────────────┐
  │  #  │                    화면 (경로)                     │  필요 카테고리  │                        근거                         │
  ├─────┼────────────────────────────────────────────────────┼─────────────────┼─────────────────────────────────────────────────────┤
  │ 3   │ 심사관리 > 파이프라인 상세 (/review/pipeline/[id]) │ dd, ir, legal   │ 서류심사·DD·인터뷰 시 IR자료, DD자료, 법무검토 필요 │
  ├─────┼────────────────────────────────────────────────────┼─────────────────┼─────────────────────────────────────────────────────┤
  │ 4   │ 백오피스 > 계약관리 상세 (미구현)                  │ contract, legal │ 계약서, 주주간계약, 투자계약 등 법무 문서           │
  ├─────┼────────────────────────────────────────────────────┼─────────────────┼─────────────────────────────────────────────────────┤
  │ 5   │ OI관리 > PoC 상세 (미구현)                         │ poc, report     │ PoC 결과 보고서, 실증 관련 문서                     │
  └─────┴────────────────────────────────────────────────────┴─────────────────┴─────────────────────────────────────────────────────┘

  ---
  마스터 스펙의 DocumentCategory → 단계 매핑

  DocumentCategory         사용 단계
  ──────────────────────── ──────────────────────────────
  ir        (IR)           소싱 ✅  →  심사 ⬜
  dd        (DD 자료)      심사 ⬜
  contract  (계약)         백오피스 ⬜
  mentoring (멘토링)       보육 ✅
  poc       (PoC)          보육 ✅  →  OI ⬜
  report    (보고서)       보육 ✅  →  OI ⬜
  legal     (법무)         심사 ⬜  →  백오피스 ⬜
  other     (기타)         소싱 ✅  (전 단계 공통)

  ✅ = 이미 해당 화면에서 사용 중 / ⬜ = 미배치

  ---
  결론

  이전 계획에서 "단계별 문서 첨부"는 다음과 같이 설계되었습니다:

  1. DocumentsTab 재사용 컴포넌트 — allowedCategories로 단계별 카테고리를 제한하는 패턴 (이미 구현)
  2. 소싱 딜 상세: IR + 기타 → 완료
  3. 보육 상세: 멘토링 + 보고서 + PoC → 완료
  4. 심사 파이프라인 상세: DD + IR + 법무 → 미구현 (review pipeline [id] 페이지에 문서 탭 없음)
  5. 백오피스 계약관리: 계약 + 법무 → 미구현 (화면 자체 미구현)
  6. OI PoC 상세: PoC + 보고서 → 미구현 (화면 자체 미구현)

  가장 시급한 것은 3번 심사 파이프라인 상세 페이지에 <DocumentsTab allowedCategories={["dd", "ir", "legal"]} /> 를 추가하는 것입니다. 이 화면은 이미 존재하기 때문에 탭만 추가하면 됩니다.
