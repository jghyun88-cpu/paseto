# 데이터 취급 규칙

> 데이터 등급별 취급 방침 및 보안 원칙

---

## 데이터 등급 분류

### Level 4: 기밀 (Confidential)

**정의**: 유출 시 법적 책임 또는 심각한 비즈니스 손실을 초래하는 정보

**예시**:
- LP 개인정보 및 출자 약정 내용
- 미체결 투자 딜의 조건 (밸류에이션, 지분율 등)
- 펀드 수익률 상세 (IRR, MOIC 등)
- LP 계약서 원문
- 포트폴리오사의 미공개 재무제표 원본

**취급 규칙**:
- 로컬 저장만 허용 (`data/` 디렉토리 내)
- 외부 API/MCP로 전송 절대 금지
- WebSearch 쿼리에 포함 금지
- 화면 공유 시 마스킹 필수
- 접근 로그 기록 필수
- 보고서 상단에 `기밀등급: 기밀` 표시

### Level 3: 대외비 (Internal Only)

**정의**: 조직 내부에서만 공유 가능한 정보

**예시**:
- 포트폴리오사의 IR 자료 (비공개)
- 내부 평가 점수 및 심사 의견
- 포트폴리오사 KPI 데이터
- 투심위 회의록
- 멘토 평가 결과

**취급 규칙**:
- 로컬 저장 및 승인된 내부 DB만 허용
- Notion, Google Sheets 등 팀 공유 도구 사용 가능 (접근 제한 설정)
- 외부 전송 시 암호화 또는 비밀번호 설정
- WebSearch에 기업명 + 미공개 정보 조합 금지
- 보고서 상단에 `기밀등급: 대외비` 표시

### Level 2: 내부용 (Internal Use)

**정의**: 팀 내 자유 공유 가능하나 외부 공개는 부적절한 정보

**예시**:
- 평가 기준표 및 루브릭
- 운영 프로세스 문서
- 내부 미팅 메모
- 시스템 설정 및 가이드

**취급 규칙**:
- 팀 도구 자유 사용 가능
- Git 저장소에 커밋 가능 (private repo)
- 외부 공개 전 확인 필요
- 보고서 상단에 `기밀등급: 내부용` 표시

### Level 1: 공개 (Public)

**정의**: 외부 공개 가능한 정보

**예시**:
- 시장 조사 보고서 (공개 데이터 기반)
- 프로그램 소개 자료
- 공개된 포트폴리오사 목록
- 뉴스/보도자료 기반 분석

**취급 규칙**:
- 제한 없음
- 출처 명시 권장

## AI 에이전트 데이터 취급 원칙

### WebSearch / WebFetch 사용 시

**허용**:
- 공개된 시장 데이터 검색
- 뉴스 기사 및 보도자료 검색
- 공개된 기업 정보 검색
- 산업 트렌드 리서치

**금지**:
- 기업명 + 미공개 재무정보 조합 검색
- LP 이름 + 투자 정보 조합 검색
- 미공개 투자 조건을 검색 쿼리에 포함
- 포트폴리오사의 내부 지표를 검색 쿼리에 포함

### MCP 서버 연동 시

**허용**:
- 승인된 Notion/Google Sheets 워크스페이스 접근
- 내부 DB 쿼리 (읽기)
- Slack 채널 알림 발송 (승인된 채널만)

**금지**:
- 미승인 외부 서비스에 데이터 전송
- DB에 검증되지 않은 데이터 쓰기 (사람 확인 필수)
- Slack 공개 채널에 기밀/대외비 정보 게시

### 파일 저장 규칙

```
data/
├── applications/     # Level 3 (대외비) - 지원서 원본
├── portfolio/        # Level 3-4 - 기업별 데이터
├── market-research/  # Level 1-2 - 시장 조사
├── templates/        # Level 2 (내부용) - 템플릿
└── reference/        # Level 2 (내부용) - 참고 자료

reports/
├── screening/        # Level 3 (대외비) - 스크리닝 결과
├── analysis/         # Level 3 (대외비) - 분석 보고서
├── portfolio/        # Level 3 (대외비) - 포트폴리오 보고
├── lp/               # Level 4 (기밀) - LP 보고서
├── investment-committee/  # Level 3-4 - 투심위 자료
└── risk/             # Level 3 (대외비) - 리스크 경보
```

## 감사 로그 정책

### 기록 대상

| 활동 | 기록 항목 |
|------|----------|
| 파일 생성/수정 | 시간, 파일경로, 에이전트/사용자 |
| 보고서 생성 | 시간, 보고서 유형, 대상 기업 |
| MCP 데이터 접근 | 시간, 서비스명, 쿼리 요약 |
| WebSearch 실행 | 시간, 검색 쿼리 |
| 에이전트 실행 | 시간, 에이전트명, 작업 요약 |

### 로그 형식

```
[YYYY-MM-DD HH:MM:SS] [유형] [주체] [대상] [액션] [비고]
```

## 데이터 폐기

### 폐기 대상 식별
- 보존 기간 만료 데이터 (compliance.md 참조)
- 프로젝트 완료 후 불필요한 임시 파일
- 탈락 지원서 (배치 종료 후 1년 경과)

### 폐기 절차
1. 폐기 대상 목록 작성
2. 담당자 확인 및 승인
3. 완전 삭제 실행
4. 폐기 기록 `logs/disposal.log`에 기록

### 임시 파일 관리
- 분석 중간 산출물은 작업 완료 후 정리
- `logs/` 디렉토리의 세션 로그는 자동 관리
- 백업 파일은 월 1회 정리
