# AI 스크리닝 보고서 작성 기준

> eLSA 시스템 AI 스크리닝 보고서의 표준 구조, 평가 항목, 채점 기준, 작성 원칙을 정의한다.
> 본 문서는 AI 에이전트(document_analyzer)의 프롬프트 및 report_generator의 출력 구조를 규정한다.

---

## 1. 보고서 구조 (9개 섹션)

| 섹션 | 제목 | 설명 |
|------|------|------|
| 1 | 요약 | 총점, 판정, 한줄 요약, 판정 근거 |
| 2 | 기본 적격성 검사 | 4개 Pass/Fail 항목 |
| 3 | 기업 개요 | 기업명, 대표자, 사업 분야, 주요 제품, 팀 규모, 투자 요청, 정부 R&D |
| 4 | 항목별 평가 | 정량 4항목 + 정성 4항목 = 8개 항목 상세 평가 |
| 5 | 핵심 강점 (Top 3) | 각 3~5문장 상세 서술 |
| 6 | 주요 우려사항 (Top 3) | 각 3~5문장 상세 서술 |
| 7 | 추가 확인 필요사항 | 2차 심사 진입 전 확인해야 할 사항 목록 |
| 8 | 종합 의견 | 5~10문장 투자 관점 종합 판단 |
| 9 | 점수 요약표 | 전 항목 점수/만점/비율 테이블 |

## 2. 기본 적격성 검사 (Pass/Fail)

2차 심사 진입 전 반드시 통과해야 하는 관문 검사. 1개라도 Fail이면 정량/정성 평가 불진행.

| 항목 | 판정 기준 |
|------|----------|
| 법적 결격사유 | 대표자/기업의 법적 결격사유, 소송, 형사 이력 확인. 불확실 시 "미확인" 표기 |
| 동일/유사 서비스 중복 지원 | 동일 배치 또는 이전 배치에 동일/유사 서비스로 지원한 이력 확인 |
| 반사회적 사업 모델 | 사회적으로 부정적 영향을 미치는 사업 모델 여부 판단 |
| 기존 포트폴리오사와 직접 경쟁 | 현재 포트폴리오 기업과 직접적 시장 경쟁 관계 여부 확인 |

## 3. 평가 항목 및 채점 기준 (100점 만점)

### 3-1. 정량 평가 (60점)

#### 팀 역량 (기본 15점)

대표/공동창업자의 도메인 경험, 핵심 인력 구성, 풀타임 전념 여부를 평가.

| 점수 구간 | 기준 |
|----------|------|
| 13~15 | 관련 산업 5년 이상 경험, 창업/EXIT 이력, 핵심 포지션(CEO/CTO/CPO) 전원 풀타임, 기술+사업 보완 구성, 산학협력 또는 어드바이저 네트워크 보유 |
| 10~12 | 관련 산업 3년 이상, 핵심 역할 대부분 풀타임, 적절한 보완성, 이전 인허가/제품 출시 이력 |
| 7~9 | 관련 경험 있으나 제한적, 일부 파트타임, 보완성 부족 영역 존재, 외부 자문 의존 |
| 4~6 | 관련 경험 부족, 핵심 역할 공백(CTO 미충원 등), 팀 구성 불안정 |
| 1~3 | 경험 매우 부족, 1인 창업, 팀 구성 계획 불명확, 핵심 인력 정보 미공개 |

**세부 체크리스트:**
- 창업팀 도메인 전문성 (해당 산업 경력, 인사이트)
- 이전 창업/EXIT 경험 유무
- 핵심 포지션 충원 현황 (CEO, CTO, CPO, CFO)
- 풀타임 전환 여부 및 시점
- 팀원 간 역량 보완성 (기술 + 비즈니스 + 도메인)
- 공동 창업자 간 지분 구조의 합리성
- 산학협력/어드바이저 네트워크 구성

**[미확인] 처리:** 팀 규모, 내부 풀타임 인력 수, CTO 유무 등이 IR에 미공개 시 반드시 [미확인]으로 표기하고 감점 사유에 명시.

---

#### 시장 규모 및 성장성 (기본 15점)

TAM/SAM/SOM 규모, 시장 성장률, 구조적 성장 동인을 평가.

| 점수 구간 | 기준 |
|----------|------|
| 13~15 | TAM 1조원 이상, 연 성장률 20% 이상, SAM/SOM 논리적 산출, 데이터 근거 충분, 글로벌 확장 가능 |
| 10~12 | TAM 5천억 이상, 연 성장률 15% 이상, SAM/SOM 근거 존재, 해외 시장 언급 |
| 7~9 | TAM 1천억 이상, 성장 시장이나 근거 부분적, SAM/SOM 가정 낙관적 |
| 4~6 | 시장 규모 작거나 정체, 근거 불충분, 성장 동인 불명확 |
| 1~3 | 시장 규모 불명확, 축소 시장, 근거 없음 |

**세부 체크리스트:**
- TAM 규모 및 산출 근거 (병상 수, 대상 비율, 단가 등 구체적 산출)
- SAM 정의의 합리성
- SOM 달성 경로 및 점유율 가정의 근거
- 시장 성장률 (수치 명시 여부)
- 구조적 성장 동인 (인구구조, 규제, 기술 변화 등)
- 글로벌 시장 규모 및 진출 가능성

---

#### 제품/기술 (기본 15점)

기술 차별성, 특허/IP 전략, 개발 단계, 인증 현황을 평가.

| 점수 구간 | 기준 |
|----------|------|
| 13~15 | 원천기술 보유, 특허 10건 이상 + PCT 해외 출원, 다층 IP 전략, 인증(GMP/ISO 등) 완료, 상용 제품 출시 |
| 10~12 | 핵심 기술 보유, 특허 5건 이상, IP 전략 존재, 주요 인증 진행 중, 시제품 완성 |
| 7~9 | 기술 개발 중, 특허 출원 중, 시제품 단계, 인증 계획 수립 |
| 4~6 | 기술 초기 단계, 특허 미출원, 프로토타입 미완성 |
| 1~3 | 기술 구현 계획만 존재, IP 전략 없음, 아이디어 수준 |

**세부 체크리스트:**
- 특허 포트폴리오 (등록/출원 건수, PCT 해외 출원)
- IP 전략 구조 (시스템/모듈/소모품/프로세스 등 다층 여부)
- 기술 개발 단계 (아이디어 → 프로토타입 → 시제품 → 상용)
- AI 기술 요소 (자체 모델 vs API 활용, 데이터 해자)
- 인증/인허가 현황 (GMP, ISO, FDA, CE 등)
- 기술 리스크 (완성도, 경쟁 기술, 대체 기술)

**딥테크 특화 기준:**
- AI/딥테크 기업의 경우 "자체 모델 개발 여부 vs API 활용 비율" 반드시 확인
- 데이터 해자(독점 데이터셋 규모) 평가 포함
- 미확인 시 [미확인]으로 표기

---

#### 비즈니스 모델 (기본 15점)

수익 구조, 단위 경제학, 검증 수준을 평가.

| 점수 구간 | 기준 |
|----------|------|
| 13~15 | 명확한 수익 모델, 검증된 단위 경제학(CAC/LTV), 높은 마진, 확장성 입증 |
| 10~12 | 수익 모델 정의, 마진 합리적, 단위 경제학 설계됨, 확장 가능성 |
| 7~9 | 수익 모델 존재하나 검증 부족, 확장성 불확실 |
| 4~6 | 수익 모델 불명확, 마진 낮음, 확장 어려움 |
| 1~3 | 수익 모델 없음, 수익화 계획 부재 |

**세부 체크리스트:**
- 수익 구조 (구독, 세션 과금, 소모품, SaaS, 라이선스 등)
- 매출총이익률(Gross Margin)
- 고객획득비용(CAC) / 고객생애가치(LTV) / Payback Period
- BOM(원가 구조) 공개 여부
- 손익분기점(BEP) 예상 시기
- 가격 결정력 및 경쟁 방어 가능성

**딥테크/하드웨어 특화:**
- 하드웨어 기업은 장치 단가(원가/판매가), 양산 단가, 초기 CAPEX 확인 필수
- 소모품 마진 구조 별도 평가
- 시제품 단계 기업은 BM 미검증을 유연하게 허용하되 감점 사유에 명시

### 3-2. 정성 평가 (40점)

#### 비전 및 확장성 (기본 10점)

비전의 명확성, 글로벌 확장 로드맵, 사회적 임팩트를 평가.

| 점수 구간 | 기준 |
|----------|------|
| 9~10 | 명확하고 설득력 있는 비전, 구체적 글로벌 로드맵(국가별 진출 순서), 제품 라인 확장 설계, 사회적 임팩트 명확 |
| 7~8 | 비전 명확, 해외 확장 계획 존재, 제품 확장 방향 언급 |
| 5~6 | 비전 존재하나 구체성 부족, 확장 계획 추상적 |
| 3~4 | 비전 모호, 확장 계획 미흡 |
| 1~2 | 비전 부재, 로컬 시장 한정 |

#### 솔루션 차별화 (기본 10점)

경쟁사 대비 차별점, 모방 장벽, 경쟁 우위의 지속성을 평가.

| 점수 구간 | 기준 |
|----------|------|
| 9~10 | 세계 최초/유일 포지셔닝, 다층 IP 해자, 인증 기반 제도적 진입 장벽, 경쟁사 대비 10x 개선 |
| 7~8 | 명확한 차별점, 특허 기반 모방 장벽, 경쟁사 대비 3x 이상 개선 |
| 5~6 | 차별점 존재하나 모방 용이, 경쟁 우위 일시적 |
| 3~4 | 기존 솔루션과 유사, 차별화 미미 |
| 1~2 | 차별점 없음, 레드오션 진입 |

**세부 체크리스트:**
- 경쟁사 분석 (기존 제품의 한계점 명시)
- 자사 솔루션의 구체적 개선점
- 특허/인증 기반 모방 장벽
- 기존 고객 네트워크 활용 가능성
- 임상 완료 전 단계면 1점 유보

#### 투자 적합성 (기본 10점)

투자 규모의 적절성, 액셀러레이터 포커스 영역 정합성, 멘토 매칭 적합성을 평가.

| 점수 구간 | 기준 |
|----------|------|
| 9~10 | 투자 요청 규모 배치 한도 이내, 포커스 영역 정합, 멘토 풀 매칭 최적, 기존 포트폴리오 시너지 |
| 7~8 | 투자 규모 합리적, 포커스 영역 부합, 멘토 매칭 가능 |
| 5~6 | 투자 규모 다소 불일치, 부분적 부합 |
| 3~4 | 투자 규모 불일치 심각, 포커스 영역 불일치 |
| 1~2 | 완전 불일치, 투자 불가 구조 |

**핵심 체크 포인트:**
- 투자 요청액 vs 배치 한도 비교 (N배 초과 시 구체적 명시)
- 리드/공동/브릿지 투자 역할 기대 확인
- 공동 투자자 협의 현황
- 밸류에이션 적정성
- 투자 요청 대비 역할 기대 불일치 시 구조적 감점 불가피

#### 프리젠테이션 (기본 10점)

IR 자료의 논리적 구성, 핵심 정보 포함 여부, 시각적 품질을 평가.

| 점수 구간 | 기준 |
|----------|------|
| 9~10 | 논리적 흐름 완벽, 핵심 정보 전부 포함(팀/시장/BM/재무/로드맵/투자조건), 수치 근거 풍부, 시각적 우수 |
| 7~8 | 논리적 흐름 양호, 대부분의 핵심 정보 포함, 수치 근거 존재 |
| 5~6 | 구성 보통, 일부 핵심 정보 누락, 근거 부분적 |
| 3~4 | 구성 미흡, 다수 핵심 정보 누락 |
| 1~2 | IR 자료 부실, 정보 불충분 |

**누락 시 감점 항목:**
- 팀 구성 상세 정보 미공개
- 밸류에이션 미공개
- 기존 제품 매출 실적 미공개
- 번레이트/런웨이 미공개
- AI 기술 구조 명세 미공개

## 4. 종합 판정 기준

| 총점 | 판정 | 권고 액션 |
|------|------|----------|
| 80~100 | **통과 (Pass)** | 2차 심사(미팅) 진행 권고 |
| 70~79 | **조건부 통과 (Conditional)** | 특정 조건 확인 후 진행 |

## 5. 작성 원칙

### 5-1. 근거 기반 원칙
- 문서에서 확인된 사실만 기재
- 추측은 반드시 [미확인] 또는 [추정]으로 표기
- 구체적 수치(금액, 인원수, 특허 건수, 시장 규모 등)를 최대한 인용
- 각 평가 항목은 반드시 **근거와 함께 3~5문장**으로 작성

### 5-2. 미확인 항목 처리
- IR 자료에 미공개된 핵심 정보는 [미확인]으로 명시
- 미확인 항목은 해당 평가 항목의 감점 사유로 반영
- 추가 확인 필요사항 섹션에 구체적으로 나열

### 5-3. 편향 방지
- 특정 산업/배경에 대한 선입견 배제
- 절대 기준으로 평가 (상대 평가 지양)
- 초기 스타트업 특성(데이터 부족 등) 감안하되, 감안 사실을 명시

### 5-4. 딥테크 특화 원칙
- 의료기기/바이오: 인허가 경로, 임상 계획, 규제 리스크 별도 평가
- AI/소프트웨어: 자체 모델 vs API 의존도, 데이터 해자 평가
- 하드웨어: BOM 원가, 양산 단가, CAPEX 구조 평가
- 시제품 단계: 상용화 전 기술 리스크를 명시하되 과도한 패널티 지양

### 5-5. 강점/우려사항 작성 기준
- 각 항목은 **구체적 근거 → 의미 해석 → 투자 관점 시사점** 순서로 서술
- 우려사항은 리스크 수준과 완화 가능성을 함께 언급
- "~할 수 있다" 식의 모호한 표현 대신 "~이므로 ~가 필요하다" 식의 구체적 표현 사용

### 5-6. 기업 웹사이트 직접 크롤링 (필수)

AI 스크리닝 실행 시 해당 기업의 공식 웹사이트를 **반드시 직접 크롤링**하여 분석에 반영한다.
IR 자료와 웹사이트 정보를 교차 검증하여 분석 신뢰도를 높인다.

#### 크롤링 대상
- 스타트업 DB에 등록된 `website` URL (Startup 모델의 `website` 필드)
- URL 미등록 시 기업명 기반 웹 검색으로 공식 사이트를 탐색

#### 수집 항목

| 수집 항목 | 설명 | 활용 평가 항목 |
|----------|------|--------------|
| 회사 소개 | About/회사 소개 페이지 내용 | 팀 역량, 비전 및 확장성 |
| 제품/서비스 설명 | 제품 페이지, 기능 목록, 기술 설명 | 제품/기술, 솔루션 차별화 |
| 팀 정보 | 팀 페이지, 경영진 프로필, 채용 현황 | 팀 역량 |
| 고객 사례/파트너십 | 레퍼런스, 고객사 로고, 제휴 현황 | 비즈니스 모델, 초기 검증 |
| 뉴스/블로그 | 최신 소식, 기술 블로그, 보도자료 | 시장 잠재력, 기술 동향 |
| 채용 공고 | 현재 채용 포지션, 규모 | 팀 역량, 성장 단계 판단 |
| 연락처/위치 | 사무실 주소, 지사 현황 | 기본 적격성, 기업 실재 확인 |

#### 교차 검증 규칙
- IR 자료에 기재된 제품/서비스가 웹사이트에 실제 존재하는지 확인
- 웹사이트 팀 정보와 IR 자료의 팀 구성이 일치하는지 비교
- 웹사이트 최종 업데이트 시점을 확인하여 사업 활성도 판단
- 불일치 발견 시 보고서의 `verification_needed`에 명시

#### 기술 요구사항
- `web_research_service.py`에서 Tavily 웹 검색과 별도로 기업 웹사이트 직접 크롤링 수행
- 크롤링 실패 시 (접속 불가, 타임아웃) 보고서에 "[웹사이트 접속 불가]" 명시
- 크롤링 결과는 LLM 분석 컨텍스트(`startup_info`)에 `## 기업 웹사이트 분석 결과` 섹션으로 추가
- robots.txt를 준수하며, 과도한 요청을 방지 (최대 10페이지, 타임아웃 10초)

#### JSON 출력 확장

기업 웹사이트 크롤링 결과는 `scores` 내 `_website_analysis` 키에 저장:

## 6. 배치별 가중치 조정

배치(Batch)마다 중점 평가 항목이 달라질 수 있다. 기본 배점은 위 표준을 따르되, 배치 설정 파일에서 다음을 조정 가능:

| 조정 가능 항목 | 범위 | 예시 |
|--------------|------|------|
| 정량 각 항목 배점 | 정량 합계 60점 유지 | 팀 역량 15→20, BM 15→7 |
| 정성 각 항목 배점 | 정성 합계 40점 유지 | 비전 10→15, 프리젠테이션 0→5 |
| Gate 기준 점수 | 변경 가능 | 통과 80→75 |
| 특별 심사 기준 추가 | 배치별 | AI 기술 구조 명세 필수 등 |

조정 시 보고서 서두에 "배치 가중치 조정 적용 안내" 테이블을 포함하여 기본 대비 변경 사항과 사유를 명시한다.

## 7. JSON 출력 스키마 (AI 에이전트용)

AI 에이전트는 반드시 아래 JSON 구조로 분석 결과를 반환해야 한다.

```json
{
  "scores": {
    "team_competency": 1-5,
    "market_potential": 1-5,
    "tech_differentiation": 1-5,
    "business_model": 1-5,
    "vision_scalability": 1-5,
    "solution_differentiation": 1-5,
    "investment_fit": 1-5,
    "problem_clarity": 1-5
  },
  "total_score": 0-100,
  "recommendation": "pass|conditional|hold|decline",
  "risk_level": "low|medium|high|critical",
  "one_line_summary": "한줄 요약",
  "eligibility_check": {
    "legal_issue": "Pass|Fail|미확인",
    "duplicate_application": "Pass|Fail|미확인",
    "antisocial_model": "Pass|Fail|미확인",
    "portfolio_conflict": "Pass|Fail|미확인"
  },
  "company_overview": {
    "company_name": "",
    "representative": "",
    "business_field": "",
    "main_product": "",
    "team_size": "",
    "investment_request": "",
    "government_rd": ""
  },
  "score_details": [
    {"item": "팀 역량", "score": "N/15", "analysis": "3~5문장 상세 평가"},
    {"item": "시장 규모 및 성장성", "score": "N/15", "analysis": "상세 평가"},
    {"item": "제품/기술", "score": "N/15", "analysis": "상세 평가"},
    {"item": "비즈니스 모델", "score": "N/15", "analysis": "상세 평가"},
    {"item": "비전 및 확장성", "score": "N/10", "analysis": "상세 평가"},
    {"item": "솔루션 차별화", "score": "N/10", "analysis": "상세 평가"},
    {"item": "투자 적합성", "score": "N/10", "analysis": "상세 평가"},
    {"item": "프리젠테이션", "score": "N/10", "analysis": "상세 평가"}
  ],
  "top_strengths": ["강점 1 (3~5문장)", "강점 2", "강점 3"],
  "top_concerns": ["우려사항 1 (3~5문장)", "우려사항 2", "우려사항 3"],
  "verification_needed": ["추가 확인 1", "확인 2", "..."],
  "overall_opinion": "5~10문장 종합 의견",
  "summary": "3~5문장 요약",
  "key_findings": ["핵심 발견 1", "핵심 발견 2", "..."]
}
```

**scores 필드 설명:**
- 각 항목 1~5점 척도 (보고서 내부에서 기본 배점으로 환산)
- 5점 척도 → 기본 배점 환산: `환산 점수 = (5점 점수 / 5) × 기본 배점`
- 예: 팀 역량 4/5 → (4/5) × 15 = 12점
