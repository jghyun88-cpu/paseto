# 배치/프로그램별 커스터마이징 규칙

> 배치(기수), 프로그램 유형별로 차별화된 평가 기준과 운영 설정을 적용하는 규칙

---

## 1. 배치 구조

### 배치 디렉토리 구조

```
data/
├── applications/
│   ├── batch-2025-H2/     # 2025년 하반기 배치
│   ├── batch-2026-H1/     # 2026년 상반기 배치
│   └── batch-2026-S/      # 2026년 특별 배치
├── batch-config/
│   ├── batch-2025-H2.md   # 배치별 설정
│   ├── batch-2026-H1.md
│   └── default.md         # 기본 설정
```

### 배치 설정 파일 형식

각 배치 설정 파일(`data/batch-config/{배치명}.md`)은 다음 구조를 따릅니다:

```markdown
---
배치명: 2026-H1
기간: 2026-01 ~ 2026-06
프로그램: 일반 (또는 특화/글로벌/임팩트)
모집규모: 10팀
투자규모: 팀당 최대 3억원
---

## 평가 가중치 조정
| 항목 | 기본 가중치 | 이 배치 가중치 | 사유 |
|------|-----------|-------------|------|
| 팀 역량 | 20점 | 25점 | 팀 역량 중시 |
| 시장 매력도 | 15점 | 15점 | 유지 |
| ... | | | |

## 우선 산업 분야
- AI/딥테크
- ESG/클린테크

## 특별 심사 기준
- [이 배치에만 적용되는 특수 기준]

## 멘토 풀
- [이 배치에 배정된 멘토 목록 또는 전체 풀 사용]
```

## 2. 프로그램 유형별 설정

### 일반 프로그램 (Default)

| 항목 | 설정 |
|------|------|
| 평가 루브릭 | evaluation-rubric.md (표준) |
| 스크리닝 통과 기준 | 80점 이상 |
| 조건부 범위 | 70-79점 |
| 투자 한도 | 팀당 최대 5억원 |
| 보육 기간 | 6개월 |
| 멘토링 | 월 2회 |
| 보고 주기 | 월간 KPI |

### 특화 프로그램 (바이오/딥테크)

| 항목 | 설정 | 변경 사유 |
|------|------|----------|
| 평가 루브릭 | 기술 검증 가중치 +10 | 기술 난이도 높음 |
| 스크리닝 통과 기준 | 75점 이상 | 초기 트랙션 어려움 감안 |
| 조건부 범위 | 65-74점 | 기술 잠재력 고려 |
| 투자 한도 | 팀당 최대 10억원 | 높은 자본 요구 |
| 보육 기간 | 12개월 | 긴 개발 주기 |
| 멘토링 | 월 4회 (기술 멘토 포함) | 전문 지원 필요 |
| 보고 주기 | 월간 KPI + 기술 마일스톤 | 기술 진척 추적 |
| 추가 평가 | 특허/IP 분석 | 기술 자산 평가 |

### 글로벌 프로그램

| 항목 | 설정 | 변경 사유 |
|------|------|----------|
| 평가 루브릭 | 글로벌 시장 진출 가중치 +10 | 해외 확장 포커스 |
| 스크리닝 통과 기준 | 80점 이상 | 표준 유지 |
| 투자 한도 | 팀당 최대 7억원 | 해외 진출 비용 |
| 보육 기간 | 9개월 | 글로벌 진출 준비 |
| 멘토링 | 월 3회 (해외 멘토 포함) | 글로벌 네트워크 |
| 보고 주기 | 월간 KPI (영문 병행) | 글로벌 LP 보고 |
| 추가 평가 | 영문 IR, 해외 레퍼런스 | 글로벌 준비도 |

### 임팩트 프로그램 (ESG/소셜)

| 항목 | 설정 | 변경 사유 |
|------|------|----------|
| 평가 루브릭 | 사회적 임팩트 항목 추가 (+15점) | 임팩트 측정 |
| 스크리닝 통과 기준 | 총점 80점 이상 (임팩트 10점 이상) | 임팩트 최소 기준 |
| 투자 한도 | 팀당 최대 5억원 | 표준 |
| 보육 기간 | 6개월 | 표준 |
| 멘토링 | 월 2회 (임팩트 전문가 포함) | 임팩트 전략 |
| 보고 주기 | 월간 KPI + 분기 임팩트 리포트 | 임팩트 추적 |
| 추가 평가 | Theory of Change, 임팩트 KPI | 사회적 가치 측정 |

## 3. 배치별 가중치 조정 규칙

### 평가 가중치 조정 방법

에이전트가 스크리닝/분석 시 다음 순서로 설정을 로드합니다:

```
1. data/batch-config/{현재배치}.md 확인
2. 존재하면 → 해당 배치 설정 적용
3. 없으면 → data/batch-config/default.md 적용
4. default도 없으면 → evaluation-rubric.md 기본 가중치 사용
```

### 가중치 조정 가능 항목

| 평가 영역 | 기본 배점 | 조정 범위 | 조정 시 주의사항 |
|-----------|----------|----------|----------------|
| 팀 역량 | 20점 | 15-30점 | 총점 100점 유지 |
| 시장 매력도 | 15점 | 10-25점 | 총점 100점 유지 |
| 제품/기술 | 15점 | 10-25점 | 총점 100점 유지 |
| 비즈니스 모델 | 10점 | 5-15점 | 총점 100점 유지 |
| 정성 평가 | 40점 | 30-50점 | 정량 + 정성 = 100 |

**중요**: 가중치를 조정하더라도 **정량(60) + 정성(40) = 100점** 총합은 유지해야 합니다. 정량/정성 비율 자체를 변경하려면 프로그램 유형별 설정에서 명시합니다.

## 4. 배치 운영 일정 템플릿

```
주요 마일스톤:
  W-8   모집 공고
  W-6   지원서 접수 시작
  W-2   지원서 접수 마감
  W-1   1차 스크리닝 (application-screener)
  W0    2차 심층 분석 (ir-analyst + market-researcher + financial-analyst)
  W+1   투심위 상정 (deal-coordinator 풀 사이클)
  W+2   투자 확정 및 계약
  W+3   보육 프로그램 시작, 멘토 매칭 (mentor-matcher)
  M+1   첫 번째 월간 리뷰
  M+3   중간 점검 (portfolio-reporter + risk-monitor)
  M+5   졸업 준비/데모데이 예비 심사
  M+6   데모데이/졸업, LP 보고 (lp-report-writer)
```

## 5. 데이터 격리 규칙

- 각 배치의 지원서는 `data/applications/{배치명}/` 하위에 저장
- 보고서 파일명에 배치 식별자를 포함 가능 (예: `{기업명}_screening_{배치}_{날짜}.md`)
- 배치 간 지원서/평가 결과는 상호 참조하지 않음 (독립 심사 원칙)
- 포트폴리오 데이터는 배치 구분 없이 통합 관리 (`data/portfolio/`)

## 6. 커스터마이징 적용 예시

### 예시: 2026-H1 바이오 특화 배치 설정

```markdown
---
배치명: 2026-H1-Bio
기간: 2026-01 ~ 2026-06
프로그램: 특화 (바이오/헬스케어)
모집규모: 5팀
투자규모: 팀당 최대 8억원
---

## 평가 가중치 조정
| 항목 | 기본 | 조정 | 사유 |
|------|------|------|------|
| 팀 역량 | 20 | 25 | 의료/과학 전문성 중시 |
| 시장 매력도 | 15 | 10 | 니치 시장 허용 |
| 제품/기술 | 15 | 20 | 기술 혁신성 강조 |
| 비즈니스 모델 | 10 | 5 | 초기 수익 모델 미확정 허용 |
| 정성 평가 | 40 | 40 | 유지 |

## 우선 산업 분야
- AI 기반 신약 개발
- 디지털 치료제 (DTx)
- 의료기기 소프트웨어 (SaMD)
- 정밀의료/진단

## 특별 심사 기준
- 임상/인허가 로드맵 필수 제출
- 핵심 연구진 구성 평가 (논문, 특허 실적)
- 규제 대응 전략 필수 포함

## 멘토 풀
- M-001 (김진혁, 바이오/헬스케어)
- M-012 (장현우, 딥테크)
- 외부 자문: [의료 전문가 추가 섭외]
```
