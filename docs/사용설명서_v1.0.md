## 1. 시스템 개요

### 1.1 eLSA란?

eLSA(딥테크 액셀러레이터 업무운영시스템)는 액셀러레이터의 전체 업무 주기를 하나의 플랫폼에서 관리하는 웹 기반 시스템입니다.

**소싱 -> 심사 -> 투자 -> 보육 -> 수요기업 연결 -> 후속투자 -> 회수**

이 모든 과정을 하나의 파이프라인으로 연결하여, 팀 간 인계가 자연스럽게 이루어지고, 모든 의사결정이 문서와 수치 기반으로 기록됩니다.

### 1.2 대상 사용자

| 역할 | 팀 | 주요 업무 |
|------|-----|----------|
| 대표/파트너 | 전체 | 투자위원회(IC) 의사결정, 전사 대시보드 |
| 심사역/애널리스트 | Sourcing팀 | 딜 발굴, 파이프라인 관리, 1차 스크리닝 |
| 심사역 | 심사팀 | 서류심사, 인터뷰, DD, 투자메모, IC 상정 |
| PM/매니저 | 보육팀 | 온보딩, 멘토링, KPI 관리, Demo Day |
| OI 매니저 | OI팀 | 수요기업 매칭, PoC, 정부사업 연계 |
| 운영 담당 | 백오피스팀 | 계약, 투자 집행, 조합 관리, 보고서 |
| 관리자 | 관리 | 사용자 관리, SOP, 양식 관리, 시스템 설정 |

### 1.3 핵심 설계 원칙

1. **단일 기업 ID**: 하나의 스타트업은 소싱부터 회수까지 같은 ID로 추적
2. **릴레이 관리**: 팀 간 인계 문서를 통해 업무가 자연스럽게 이어짐
3. **문서/KPI 기반 의사결정**: 모든 판단은 기록과 수치에 근거
4. **자동화**: 10가지 핵심 자동화로 반복 업무 최소화

## 2. 로그인 및 인증

### 2.1 로그인 방법

1. 웹 브라우저에서 시스템 주소에 접속합니다
2. 이메일과 비밀번호를 입력합니다
3. "로그인" 버튼을 클릭합니다

### 2.2 권한 체계

시스템은 **역할(Role)** 과 **팀(Team)** 기반으로 접근 권한을 제어합니다.

| 역할 | 설명 |
|------|------|
| admin | 전체 관리 권한 |
| partner | 투자위원회 + 전체 열람 |
| analyst | 심사/분석 권한 |
| pm | 보육 관리 권한 |
| oi_manager | 오픈이노베이션 관리 |
| backoffice | 백오피스 운영 |

**권한 제한 규칙:**
- Sourcing팀은 최종 투자조건을 결정할 수 없습니다
- 보육팀은 투자 약속/밸류에이션 협상을 하지 않습니다
- 백오피스는 투자 판단/성장전략 수립을 하지 않습니다

## 3. 화면 구성

### 3.1 전체 레이아웃

eLSA는 K-SENS II 스타일의 3단 레이아웃을 사용합니다.

```
+------------------------------------------------------------------+
|  상단 GNB (전체업무 | 소싱관리 | 심사관리 | 보육관리 | OI관리 | 백오피스 | KPI | 관리)  |
+------+-------------------+---------------------------------------+
| 좌측  | 트리 메뉴         |                                       |
| 툴바  | (현재 탭에 따라    |        메인 컨텐츠 영역               |
|       |  동적으로 변경)    |                                       |
|       |                   |                                       |
+------+-------------------+---------------------------------------+
```

### 3.2 상단 GNB (Global Navigation Bar)

화면 상단의 탭을 클릭하여 각 모듈로 이동합니다.

| 탭 | 설명 | 주요 하위 메뉴 |
|----|------|-------------|
| 전체업무 | 전사 대시보드 | 대시보드, 스타트업관리 |
| 소싱관리 | Sourcing팀 업무 | 딜플로우관리, 스크리닝관리, 인계, 소싱분석 |
| 심사관리 | 심사팀 업무 | 파이프라인, 서류심사, 인터뷰, DD, 투자메모, IC |
| 보육관리 | 보육팀 업무 | 포트폴리오, 온보딩, 멘토링, KPI, Demo Day |
| OI관리 | OI팀 업무 | 수요맵, 매칭, PoC, 전환, 정부사업 |
| 백오피스 | 백오피스 업무 | 계약, 집행, Cap Table, 조합, 보고서 |
| KPI | 성과 관리 | 팀별 KPI(5종), 전사 대시보드 |
| 관리 | 시스템 관리 | SOP, 양식, JD, 사용자 |

### 3.3 좌측 트리 메뉴

현재 선택된 GNB 탭에 따라 좌측에 트리 메뉴가 표시됩니다. 메뉴 항목을 클릭하면 해당 페이지로 이동합니다.

## 4. 전체업무 모듈

### 4.1 대시보드

시스템 접속 시 가장 먼저 표시되는 화면입니다.

**표시 정보:**
- 전체 딜 파이프라인 현황 (단계별 건수)
- 최근 등록 스타트업
- 주요 알림
- 팀별 핵심 KPI 요약

### 4.2 스타트업 관리

전체 스타트업 마스터 데이터를 관리하는 영역입니다.

**스타트업 목록** (`/startup`)
- 전체 등록 스타트업 목록 표시
- 검색, 필터링 (업종, 단계, 상태) 지원
- 기업명 클릭 시 상세 정보 페이지로 이동

**스타트업 등록** (`/startup/new`)
- 기업 기본정보 입력 (기업명, 대표자, 법인번호, 사업자등록번호)
- 사업정보 (산업분류, 주요제품, 상장시장)
- 재무정보 (자산총액, 자본금, 매출액, 영업이익)
- 연구개발 (기술연구소, 연구인력)
- 연락처 정보 (도시, 웹사이트, 담당자, 연락처, 이메일)

**스타트업 상세** (`/startup/[id]`)
- 13개 탭으로 구성된 통합 프로필
- 기본정보, 딜플로우, 스크리닝, 심사, 투자, 보육, KPI 등

## 5. 소싱관리 모듈

소싱팀의 핵심 업무인 딜 발굴, 파이프라인 관리, 스크리닝, 심사팀 인계를 지원합니다.

### 5.1 딜플로우 칸반보드

**경로:** 소싱관리 > 딜플로우관리 > 딜플로우 칸반보드

칸반보드 형식으로 딜의 진행 상태를 시각적으로 관리합니다.

**단계:**
```
유입 -> 1차 스크리닝 -> 심층검토 -> 인터뷰 -> 기초실사
-> IC 대기 -> IC 심사중 -> 승인/조건부/보류/부결
-> 계약 -> 클로징 -> 포트폴리오
```

**사용 방법:**
1. 칸반 카드를 드래그하여 다음 단계로 이동
2. 카드 클릭 시 해당 딜 상세 정보 표시
3. 각 단계별 건수가 열 헤더에 표시

### 5.2 딜 목록

**경로:** 소싱관리 > 딜플로우관리 > 딜 목록

테이블 형식으로 모든 딜을 조회합니다.

| 열 | 설명 |
|----|------|
| 기업명 | 클릭 시 딜 상세 페이지로 이동 |
| 섹터 | 산업 분류 |
| 단계 | 투자 단계 (Pre-seed, Seed 등) |
| 발굴경로 | 딜 소싱 채널 |
| 발굴자 | 담당 심사역 |
| 상태 | 현재 딜 진행 단계 |
| 등록일 | 딜 등록일 |

**검색/필터:**
- 검색: 기업명으로 검색
- 상태 필터: 드롭다운으로 특정 단계만 조회

### 5.3 딜 등록

**경로:** 소싱관리 > 딜플로우관리 > 딜 등록

새로운 딜을 시스템에 등록합니다.

**등록 절차:**
1. 기업(고객)명 검색 (스타트업 관리에 먼저 등록 필요)
2. 검색 결과에서 기업 선택
3. 발굴일자, 섹터, 단계, 발굴경로 입력
4. 발굴자 선택
5. 한줄 소개 및 상세 설명 작성
6. "등록" 버튼 클릭

**주의사항:**
- 스타트업 관리에 기업이 먼저 등록되어 있어야 합니다
- 등록 시 자동으로 DealFlow "유입" 단계가 생성됩니다

### 5.4 딜 상세

**경로:** 딜 목록에서 기업명 클릭

딜의 모든 정보를 한 화면에서 확인할 수 있습니다.

**탭 구성:**
- **기본정보**: 딜 정보, 기업 기본정보, 사업정보, 재무정보, 연구개발, 연락처
- **문서**: 관련 문서 관리 (준비 중)
- **미팅**: 미팅 기록 관리 (준비 중)

**주요 기능:**
- **수정 버튼**: 딜 수정 페이지로 이동 (발굴정보 편집)
- **삭제 버튼**: 딜 삭제 (확인 대화상자 표시)

**표시 섹션:**
1. 기업 발굴정보: 기업명, 발굴일자, 섹터, 단계, 발굴경로, 발굴자, 상세설명
2. 기업 기본정보: 기업명, 설립일자, 법인번호, 사업자번호, 대표자, 임직원수, 소재지
3. 사업정보: 표준산업분류, 주요사업, 상장시장, 상장일자
4. 재무정보: 자산총액, 자본금, 매출액, 영업이익
5. 연구개발: 기술연구소 유무, 연구인력
6. 연락처 정보: 도시, 웹사이트, 실무담당, 연락처, 이메일
7. 딜플로우 이력: 단계 이동 기록

### 5.5 딜 수정

**경로:** 딜 상세 페이지에서 "수정" 버튼 클릭

딜의 발굴정보를 수정합니다.

**수정 가능 항목:**
- 기업(고객)명 (기업 변경 가능)
- 발굴일자
- 섹터
- 단계
- 발굴경로
- 발굴자
- 한줄 소개
- 상세 설명

**저장 방법:**
1. 원하는 항목을 수정합니다
2. "수정" 버튼을 클릭합니다
3. 성공 시 딜 상세 페이지로 자동 이동합니다

### 5.6 1차 스크리닝

**경로:** 소싱관리 > 스크리닝관리 > 1차 스크리닝 작성

**평가 항목 (SRC-F02):**
- 풀타임 전념 여부
- 문제 정의 명확성
- 기술 차별화
- 시장 잠재력
- 초기 검증
- 법적 결격사유
- 전략 적합성
- 종합 점수 (자동 계산)
- 추천: 통과/보류/탈락

### 5.7 심사팀 인계

**경로:** 소싱관리 > 심사팀 인계

1차 스크리닝 통과 건을 심사팀에 인계합니다.

**인계 패키지 포함 사항:**
- 기업 기본정보
- 스크리닝 결과
- 발굴 배경 및 추천 사유
- 추가 참고자료

## 6. 심사관리 모듈

심사팀의 서류심사, 인터뷰, DD, 투자메모 작성, IC 상정을 지원합니다.

### 6.1 심사 파이프라인

**경로:** 심사관리 > 심사 파이프라인

칸반 형식으로 심사 진행 현황을 관리합니다.

**단계:** 심층검토 -> 인터뷰 -> DD -> IC대기 -> IC심의

### 6.2 서류심사 (5축 평가)

**경로:** 심사관리 > 서류심사

**5축 평가 항목:**
1. 팀 역량 (Team Score)
2. 문제/솔루션 (Problem Score)
3. 솔루션 차별화 (Solution Score)
4. 시장 기회 (Market Score)
5. 트랙션 (Traction Score)

**3축 추가 평가:**
- 숫자 문해력 (Number Literacy)
- 고객 경험 (Customer Experience)
- 기술 해자 (Tech Moat)

### 6.3 구조화 인터뷰

**경로:** 심사관리 > 인터뷰

체계적인 인터뷰를 진행하고 결과를 기록합니다.

### 6.4 DD (실사) 체크리스트

**경로:** 심사관리 > DD

13개 DD 항목을 체크합니다:
- 사업계획서, 재무제표, 법인등기부, 주주명부, 지적재산권
- 핵심계약서, 인허가, 소송현황, 기술검증, 시장보고서
- 고객레퍼런스, 팀이력, 제품데모

### 6.5 투자메모 작성

**경로:** 심사관리 > 투자메모 작성

**메모 섹션 (INV-F02):**
1. 기업 개요 (Overview)
2. 팀 평가 (Team Assessment)
3. 시장 평가 (Market Assessment)
4. 기술/제품 평가 (Tech/Product Assessment)
5. 트랙션 (Traction)
6. 리스크 분석 (Risks)
7. 우리가 줄 수 있는 가치 (Value-Add Points)
8. 투자 조건 제안 (Proposed Terms)

### 6.6 IC (투자위원회)

**경로:** 심사관리 > IC

IC 안건을 상정하고, 결정을 기록합니다.

**IC 결정 유형:**
- 승인 (Approved)
- 조건부 승인 (Conditional)
- 보류 (On Hold)
- 보육 우선 (Incubation First)
- 부결 (Rejected)

**자동화:** IC 승인 시 계약 워크플로우(Contract) + 보육 온보딩(Incubation) 자동 생성

## 7. 보육관리 모듈

포트폴리오 기업의 성장을 지원하는 보육팀 업무를 관리합니다.

### 7.1 포트폴리오 대시보드

**경로:** 보육관리 > 포트폴리오

전체 포트폴리오 기업 현황을 한눈에 파악합니다.

**표시 정보:**
- 기업별 등급 (A/B/C/D)
- 핵심 KPI 요약
- 위기 신호 알림

### 7.2 온보딩

**경로:** 보육관리 > 온보딩

신규 포트폴리오 기업의 온보딩을 관리합니다.

**온보딩 항목 (PRG-F01):**
- 프로그램 기간 설정
- 담당 PM 배정
- 초기 진단 (Diagnosis)
- 성장 병목 파악
- 90일 액션플랜 수립

### 7.3 멘토링

**경로:** 보육관리 > 멘토링

멘토링 세션을 기록하고 추적합니다.

**세션 기록 (PRG-F03):**
- 멘토 정보
- 세션 일자/주제
- 사전 어젠다
- 논의 요약
- 피드백
- 액션 아이템 (완료율 추적)
- 다음 세션 예정일

**자동화:** 액션아이템 등록 시 Task 자동 생성 + 기한 알림

### 7.4 KPI 관리

**경로:** 보육관리 > KPI 입력 / KPI 추이

월간 KPI를 입력하고 추이를 분석합니다.

**추적 지표 (PRG-F04):**
| 지표 | 설명 |
|------|------|
| 매출 (Revenue) | 월 매출액 |
| 고객 수 (Customer Count) | 유료 고객 수 |
| 활성 사용자 (Active Users) | MAU |
| PoC 건수 | 진행 중인 PoC |
| 재구매율 (Repurchase Rate) | 재구매 비율 |
| 출시 속도 (Release Velocity) | 제품 출시 주기 |
| CAC | 고객획득비용 |
| LTV | 고객생애가치 |

**자동화:** KPI 3개월 연속 하락 시 위기 알림 + 등급 재검토

### 7.5 Demo Day

**경로:** 보육관리 > Demo Day

Demo Day 이벤트를 관리하고, 투자자 미팅을 추적합니다.

### 7.6 멘토 풀

**경로:** 보육관리 > 멘토 풀

멘토 프로필을 관리합니다.

**멘토 유형:**
- 전담 멘토 (Dedicated)
- 기능 멘토 (Functional)
- 산업 멘토 (Industry)
- 투자 멘토 (Investment)

## 8. OI관리 모듈

오픈이노베이션팀의 수요기업 매칭, PoC, 정부사업 연계를 관리합니다.

### 8.1 파트너 수요맵

**경로:** OI관리 > 파트너 수요맵

수요기업의 니즈를 정리하고 후보 스타트업을 매칭합니다.

**수요 정보 (OI-F01):**
- 수요기업명, 담당자
- 수요 유형 (기술도입, 공동개발, 투자 등)
- 기술 요구사항
- 타임라인, 예산 범위
- NDA 필요 여부
- 후보 스타트업 목록

### 8.2 PoC 프로젝트

**경로:** OI관리 > PoC

PoC 프로젝트를 생성하고 진행 상태를 추적합니다.

**PoC 상태:**
```
수요 파악 -> 매칭 -> 기획 -> 킥오프 -> 진행중
-> 중간점검 -> 완료 -> 상업계약/공동개발/전략투자/재시도/종료
```

**자동화:** PoC 전환가능성 "높음" 판정 시 심사팀에 전략투자 검토 알림

### 8.3 정부사업 연계

**경로:** OI관리 > 정부사업

포트폴리오 기업의 정부 지원사업 참여를 관리합니다.

### 8.4 후속투자 / EXIT

**경로:** OI관리 > 후속투자 / EXIT

후속 투자 유치 전략과 EXIT 기록을 관리합니다.

## 12. 업무 프로세스 흐름

### 12.1 전체 업무 흐름도

```
[1단계: 딜 소싱]
  Sourcing팀이 딜을 발굴하고 시스템에 등록
  -> 스타트업 정보 등록 (/startup/new)
  -> 딜 등록 (/sourcing/deals/new)
  -> 칸반보드에서 "유입" 단계로 표시
      |
      v
[2단계: 1차 스크리닝]
  Sourcing팀이 기본 적격성을 평가 (SRC-F02)
  -> 1차 스크리닝 작성 (/sourcing/screening/new)
  -> 통과/보류/탈락 판정
  -> A등급 + 인계=Y -> 자동으로 인계 패키지 생성
      |
      v
[3단계: 심사팀 인계]
  인계 패키지를 심사팀에 전달
  -> 인계 패키지 생성 (/sourcing/handover/new)
  -> 심사팀에 알림 자동 발송
  -> 심사팀 확인(Acknowledge) 시 인계 완료
      |
      v
[4단계: 심층 심사]
  심사팀이 다단계 심사 진행
  -> 서류심사: 5축 + 3축 평가 (/review/document)
  -> 구조화 인터뷰 (/review/interview)
  -> DD 체크리스트: 13개 항목 (/review/dd)
  -> 투자메모 작성: 7개 섹션 (/review/memo/new)
      |
      v
[5단계: IC 상정/결정]
  투자위원회에 안건 상정
  -> IC 안건 등록 (/review/ic)
  -> IC 결정: 승인/조건부/보류/보육우선/부결
  -> 승인 시 자동으로:
     - Contract (계약 워크플로우) 생성
     - Incubation (보육 온보딩) 생성
      |
      v
[6단계: 투자 계약]
  백오피스팀이 계약 실행
  -> 계약 관리 (/backoffice/contracts)
  -> 텀시트 -> 법률검토 -> 서명 -> 집행
  -> 17개 체크 항목 완료 시 클로징
  -> Cap Table 자동 업데이트
      |
      v
[7단계: 보육 프로그램]
  보육팀이 성장 지원
  -> 온보딩 (/incubation/onboarding)
  -> 90일 액션플랜 수립
  -> 멘토 매칭 및 멘토링 세션
  -> 월간 KPI 입력/추이 분석
  -> KPI 3개월 연속 하락 -> 위기 알림
      |
      v
[8단계: 수요기업 연결]
  OI팀이 수요기업과 매칭
  -> 파트너 수요 파악 (/oi/partners)
  -> 스타트업-수요기업 매칭
  -> PoC 프로젝트 진행
  -> 전환 결과 추적 (상업계약/전략투자 등)
      |
      v
[9단계: 후속투자/EXIT]
  후속 라운드 유치 또는 EXIT
  -> 후속투자 전략 수립 (/oi/follow-on)
  -> 투자자 미팅 추적
  -> EXIT 기록 (/oi/exits)
  -> LP 보고서 생성 (/backoffice/funds/report)
```

### 12.2 자동화 규칙 (10가지)

| # | 트리거 | 자동 액션 |
|---|--------|----------|
| 1 | 기업 등록 (SRC-F01) | Startup + DealFlow "유입" 자동 생성 |
| 2 | 스크리닝 A등급 + 인계=Y | Handover 자동 생성 + 심사팀 알림 |
| 3 | DD 13개 항목 전체 수령 | DD 완료 상태 자동 전환 |
| 4 | IC 승인 (INV-F03) | Contract + Incubation 동시 자동 생성 |
| 5 | 계약 17개 항목 완료 | Contract "클로징" + Cap Table 업데이트 |
| 6 | 멘토링 액션아이템 등록 | Task 자동 등록 + 기한 알림 |
| 7 | KPI 3개월 연속 하락 | 위기 알림 + 포트폴리오 등급 재검토 |
| 8 | PoC 전환가능성 "높음" | 심사팀에 전략투자 검토 알림 |
| 9 | 보고 마감 7/3/당일 | 자동 리마인더 발송 |
| 10 | 모든 양식 제출 | ActivityLog에 자동 기록 |

## 13. AI 에이전트 활용

eLSA는 MCP(Model Context Protocol) 서버를 통해 AI 에이전트와 연동됩니다.

### 13.1 사용 가능한 AI 에이전트

| 에이전트 | 기능 | 사용 시점 |
|---------|------|----------|
| application-screener | 지원서 자동 스크리닝 | 지원서 접수 후 1차 평가 |
| ir-analyst | IR 자료 심층 분석 | 심사 단계에서 IR 분석 |
| financial-analyst | 재무 건전성 평가 | DD 단계 재무 분석 |
| market-researcher | 시장/경쟁 분석 | 시장 조사 필요 시 |
| portfolio-reporter | 포트폴리오 현황 보고 | 정기 보고서 생성 |
| deal-review | 투자 심사 풀 사이클 | 스크리닝~투자메모 자동 |
| mentor-match | 멘토 매칭 | 온보딩 시 적합 멘토 추천 |
| risk-alert | 리스크 경보 | 위기 신호 자동 감지 |

### 13.2 AI 보고서

AI가 생성한 보고서는 `ai-agent/reports/` 디렉토리에 저장됩니다.

- `reports/screening/` - 스크리닝 결과
- `reports/analysis/` - 분석 보고서
- `reports/investment-committee/` - 투심위 자료

**주의:** AI 산출물은 반드시 담당자 검토 후 최종 확정해야 합니다.

## 14. FAQ / 문제 해결

### Q1. 로그인이 안됩니다
- 이메일과 비밀번호를 정확히 입력했는지 확인
- 계정이 비활성화되었을 수 있으므로 관리자에게 문의

### Q2. 딜 수정이 안됩니다
- 딜 수정은 해당 딜의 상세 페이지에서 "수정" 버튼을 클릭
- 수정 권한이 있는 역할/팀에 소속되어야 합니다

### Q3. 칸반보드에서 드래그가 안됩니다
- 단계 이동은 유효한 전환 경로만 허용됩니다 (예: 유입 -> 1차 스크리닝은 가능하지만, 유입 -> IC심사는 불가)

### Q4. 화면이 업데이트되지 않습니다
- 브라우저 강력 새로고침: Ctrl + Shift + R
- 캐시 삭제 후 다시 시도

### Q5. 에러 메시지가 표시됩니다
- "405 Method Not Allowed": 시스템 관리자에게 문의
- "401 Unauthorized": 로그인 세션 만료. 다시 로그인
- "404 Not Found": 해당 데이터가 존재하지 않거나 삭제됨
