# eLSA 챔피언 데모 가이드

## 데모 개요

| 항목 | 내용 |
|------|------|
| 목적 | 1차 배포 전 실사용자 검증 |
| 대상 | 소싱팀 1명 + 심사팀 1명 |
| 소요 시간 | ~1시간 |
| 승인자 | 대표 또는 파트너(IC) |

## 사전 준비

1. **프로덕션 서버 실행**
   ```bash
   docker compose -f docker-compose.prod.yml up -d --build
   ```

2. **테스트 계정 생성** (관리자 페이지에서)
   - 소싱팀 계정: `sourcing@test.com` (role: analyst, team: sourcing)
   - 심사팀 계정: `reviewer@test.com` (role: analyst, team: review)

3. **샘플 데이터 임포트** (선택)
   - CSV 파일 준비 → `/api/v1/import/startups` 로 업로드
   - 또는 데모 중 수동 등록

## 데모 시나리오: "AI 반도체 스타트업 A사"

### Scene 1: 로그인 + 대시보드 (3분)
- [ ] 소싱팀 계정으로 로그인
- [ ] 대시보드 화면 확인 (비어 있으면 EmptyState 표시)
- **확인 포인트**: 로그인 정상? 대시보드 레이아웃 괜찮은지?

### Scene 2: 딜 등록 (5분)
- [ ] 소싱 → 딜 관리 → 새 딜 등록
- [ ] "퀀텀칩 테크놀로지" 기업 정보 입력
  - 기업명: 퀀텀칩 테크놀로지
  - 대표자: 김양자
  - 산업: AI 반도체
  - 단계: Seed
  - 소싱채널: 대학연구실
- [ ] 등록 후 딜 목록에서 확인
- **확인 포인트**: 입력 과정이 직관적인지? 빠진 필드가 있는지?

### Scene 3: 칸반 파이프라인 (3분)
- [ ] 소싱 → 파이프라인 칸반 이동
- [ ] Inbound 컬럼에 등록한 딜 확인
- [ ] 드래그로 "1차 스크리닝" 컬럼으로 이동
- **확인 포인트**: 칸반 UI가 업무 흐름에 맞는지?

### Scene 4: 스크리닝 작성 (5분)
- [ ] 딜 상세 → 스크리닝 작성
- [ ] 점수 입력: 기술 85, 시장 75, 팀 80
- [ ] 등급: Pass, 메모: "핵심 기술 경쟁력 우수"
- [ ] 스크리닝 완료
- **확인 포인트**: 스크리닝 양식이 실제 업무에 맞는지?

### Scene 5: 인계 자동 생성 확인 (5분)
- [ ] 스크리닝 완료 후 → 인계 문서 자동 생성 확인
- [ ] 심사팀 계정으로 전환
- [ ] 인계 수신함에서 새 인계 확인
- [ ] 인계 상세 내용 확인 (스크리닝 결과 포함)
- [ ] 수신확인(acknowledge) 클릭
- **확인 포인트**: 인계 문서가 충분한 정보를 담고 있는지? 수신확인 후 다음 단계로 자동 전환되는지?

### Scene 6: 심사 진행 (5분)
- [ ] 심사 → 서류심사에서 해당 기업 확인
- [ ] 심사 의견 작성
- **확인 포인트**: 심사 화면이 업무에 맞는지?

### Scene 7: 제외 기능 확인 (2분)
- [ ] 보육, OI, KPI 메뉴 접근 → "준비 중" 표시 확인
- **확인 포인트**: "준비 중" 안내가 적절한지?

### Scene 8: 모바일 확인 (2분)
- [ ] 휴대폰으로 동일 URL 접속
- [ ] 대시보드, 딜 목록 확인
- **확인 포인트**: 모바일에서 기본 조회가 가능한지?

## 합격 기준

- [ ] 킬러 데모 시나리오(Scene 1-6) 중단 없이 완주
- [ ] 인계 문서 자동 생성 + 수신확인 정상 작동
- [ ] "이것 없으면 안 된다" 항목이 모두 존재
- [ ] P1 버그(기능 차단 수준) 0건

## 피드백 수집

데모 후 아래 3가지만 물어보세요:

### 1. "이것 없으면 안 된다" (Must-Have)
> 이 시스템에서 반드시 있어야 하는데 없는 것이 있나요?

### 2. "이건 필요 없다" (Remove/Simplify)
> 불필요하거나 혼란스러운 기능이 있나요?

### 3. "이게 빠졌다" (Missing)
> 업무에 꼭 필요한데 아예 없는 기능이 있나요?
