## 1. 시스템 개요

### 1.1 eLSA란?

eLSA(딥테크 액셀러레이터 업무운영시스템)는 액셀러레이터의 전체 업무 주기를 하나의 플랫폼에서 관리하는 웹 기반 시스템입니다.

**소싱 → 심사 → 투자 → 보육 → 수요기업 연결 → 후속투자 → 회수**

이 모든 과정을 하나의 파이프라인으로 연결하여, 팀 간 인계가 자연스럽게 이루어지고, 모든 의사결정이 문서와 수치 기반으로 기록됩니다.

### 1.2 대상 사용자

| 역할 | 팀 | 주요 업무 |
|------|-----|----------|
| 대표/파트너 | 전체 | 투자위원회(IC) 의사결정, 전사 대시보드 |
| 심사역/애널리스트 | Sourcing팀 | 딜 발굴, 파이프라인 관리, 1차 스크리닝 |
| 심사역 | 심사팀 | 서류심사, 인터뷰, DD, 투자메모, IC 상정 |
| PM/매니저 | 보육팀 | 온보딩, 멘토링, KPI 관리, Demo Day |
| OI 매니저 | OI팀 | 수요기업 매칭, PoC, 정부사업 연계 |
| 운영 담당 | 백오피스팀 | 계약, 투자 집행, 조합/LP 관리, 보고서 |
| 관리자 | 관리 | 사용자 관리, SOP, 양식 관리, 시스템 설정 |

### 1.3 핵심 설계 원칙

1. **단일 기업 ID**: 하나의 스타트업은 소싱부터 회수까지 같은 ID로 추적
2. **릴레이 관리**: 팀 간 인계 문서를 통해 업무가 자연스럽게 이어짐
3. **문서/KPI 기반 의사결정**: 모든 판단은 기록과 수치에 근거
4. **자동화**: 10가지 핵심 자동화로 반복 업무 최소화
5. **PoC→계약→매출 전환**: 소개가 아닌 거래와 실증으로 연결
6. **백오피스 = 리스크 통제**: 행정지원이 아닌 투자·계약·컴플라이언스 통제 허브
7. **회수 전략은 투자 시점부터 설계**: 딥테크는 투자부터 exit 경로를 가정

## 2. 로그인 및 인증

### 2.1 로그인 방법

1. 웹 브라우저에서 시스템 주소에 접속합니다
   - 로컬: `http://localhost:3000`
   - 외부: `https://winlsa.com`
2. 이메일과 비밀번호를 입력합니다
3. "로그인" 버튼을 클릭합니다

### 2.2 기본 테스트 계정

| 이메일 | 역할 | 팀 |
|--------|------|-----|
| admin@winlsa.com | admin | 전체 |
| sourcing@winlsa.com | analyst | sourcing |
| review@winlsa.com | analyst | review |
| pm@winlsa.com | pm | incubation |
| oi@winlsa.com | oi_manager | oi |
| ops@winlsa.com | backoffice | backoffice |
| partner@winlsa.com | partner | review |

### 2.3 권한 체계

시스템은 **역할(Role)** 과 **팀(Team)** 기반으로 접근 권한을 제어합니다.

| 역할 | 설명 |
|------|------|
| admin | 전체 관리 권한 |
| partner | 투자위원회 + 전체 열람 + 법적 약속 승인 |
| analyst | 심사/분석 권한 |
| pm | 보육 관리 권한 |
| oi_manager | 오픈이노베이션 관리 |
| backoffice | 백오피스 운영 |

**권한 제한 규칙:**
- Sourcing팀은 최종 투자조건을 결정할 수 없습니다
- 심사팀은 계약서 최종 보관/관리를 단독으로 하지 않습니다
- 보육팀은 투자 약속/밸류에이션 협상을 하지 않습니다
- OI팀은 법적 구속력 있는 거래 약속을 단독으로 하지 않습니다
- 백오피스는 투자 판단/성장전략 수립을 하지 않습니다

### 2.4 세션 관리

- JWT 토큰 기반 인증 (24시간 유효)
- 세션 만료 시 자동으로 로그인 페이지로 이동
- 401 에러 발생 시 토큰이 자동 갱신되지 않으므로 재로그인 필요

## 3. 화면 구성

eLSA는 K-SENS II 스타일의 3단 레이아웃을 사용합니다.

```
+------------------------------------------------------------------+
|  상단 GNB (전체업무 | 소싱관리 | 심사관리 | 보육관리 | OI관리 | 백오피스 | KPI | 관리)  |
+------+-------------------+---------------------------------------+
| 좌측  | 트리 메뉴         |                                       |
| 툴바  | (현재 탭에 따라    |        메인 컨텐츠 영역               |
|       |  동적으로 변경)    |                                       |
|       |                   |                                       |
+------+-------------------+---------------------------------------+
```

### 3.2 상단 GNB (Global Navigation Bar)

화면 상단의 탭을 클릭하여 각 모듈로 이동합니다.

| 탭 | 설명 | 페이지 수 | 주요 하위 메뉴 |
|----|------|----------|-------------|
| 전체업무 | 전사 대시보드 | 6 | 대시보드, 스타트업관리(목록/등록/상세/편집) |
| 소싱관리 | Sourcing팀 업무 | 13 | 딜플로우관리(칸반/목록/등록/상세/편집), 스크리닝, 인계, 분석 |
| 심사관리 | 심사팀 업무 | 8 | 파이프라인, 서류심사, 인터뷰, DD, 투자메모, IC |

| OI관리 | OI팀 업무 | 10 | 수요맵, 매칭, PoC, 전환, 정부사업, EXIT, 후속투자 |

## 4. 전체업무 모듈

### 4.1 대시보드

시스템 접속 시 가장 먼저 표시되는 화면입니다.

**표시 정보:**
- 전체 딜 파이프라인 현황 (단계별 건수)
- 최근 등록 스타트업
- 주요 알림
- 팀별 핵심 KPI 요약
- 타임라인 (주요 이벤트)

### 4.2 스타트업 관리

전체 스타트업 마스터 데이터를 관리하는 영역입니다.

**스타트업 목록** (`/startup`)
- 전체 등록 스타트업 목록 표시
- 검색, 필터링 (업종, 단계, 상태, 포트폴리오 여부) 지원
- 기업명 클릭 시 상세 정보 페이지로 이동

**스타트업 등록** (`/startup/new`)
- 기업 기본정보 입력 (기업명, 대표자, 법인번호, 사업자등록번호)
- 사업정보 (산업분류(KSIC), 주요제품, 상장시장, 상장일자)
- 재무정보 (자산총액, 자본금, 매출액, 영업이익)
- 연구개발 (기술연구소 유무, 연구인력 수)
- 연락처 정보 (도시, 웹사이트, 담당자, 연락처, 이메일)
- **v1.5 신규**: 법인등록번호(13자리), 사업자등록번호(10자리) 자동 검증

**스타트업 상세** (`/startup/[id]`)
- 13개 탭으로 구성된 통합 프로필:
  1. 기본정보 2. 딜플로우 3. 스크리닝 4. 서류심사 5. 인터뷰
  6. DD 7. 투자메모 8. IC결정 9. 계약 10. 보육
  11. KPI 12. 멘토링 13. 활동로그

**스타트업 편집** (`/startup/[id]/edit`)
- BHV 스타일 편집 화면
- 모든 필드 수정 가능 (API: PATCH)

## 5. 소싱관리 모듈

소싱팀의 핵심 업무인 딜 발굴, 파이프라인 관리, 스크리닝, 심사팀 인계를 지원합니다.

### 5.1 딜플로우 칸반보드

**경로:** 소싱관리 > 딜플로우관리 > 딜플로우 칸반보드

칸반보드 형식으로 딜의 진행 상태를 시각적으로 관리합니다.

**DealStage 상태 머신 (15단계):**
```
유입(inbound) → 1차스크리닝(first_screening) → 심층검토(deep_review)
→ 인터뷰(interview) → 기초실사(due_diligence) → IC대기(ic_pending)
→ IC심사중(ic_review) → 승인(approved) / 조건부(conditional) / 보류(on_hold)
  / 보육우선(incubation_first) / 부결(rejected)
→ 계약(contract) → 클로징(closed) → 포트폴리오(portfolio)
```

**사용 방법:**
1. 칸반 카드를 드래그하여 다음 단계로 이동 (유효한 전환만 허용)
2. 카드 클릭 시 해당 딜 상세 정보 표시
3. 각 단계별 건수가 열 헤더에 표시

### 5.2 딜 목록 (소싱 CRM)

**경로:** 소싱관리 > 딜플로우관리 > 딜 목록

테이블 형식으로 모든 딜을 조회합니다.

| 열 | 설명 |
|----|------|
| 기업명 | 클릭 시 딜 상세 페이지로 이동 |
| 섹터 | 산업 분류 |
| 단계 | 투자 단계 (Pre-seed, Seed 등) |
| 발굴경로 | 딜 소싱 채널 (9종) |
| 발굴자 | 담당 심사역 |
| 상태 | 현재 딜 진행 단계 (색상 코딩) |
| 등록일 | 딜 등록일 |

**소싱 채널 (9종):** 대학/연구소, CVC/OI, 기존 포트폴리오, 외부 VC, 공공기관, 경진대회, 온라인/IR, 직접발굴, 기술전시회

**검색/필터:**
- 검색: 기업명으로 검색
- 상태 필터: 드롭다운으로 특정 단계만 조회
- 채널 필터: 소싱 채널별 필터링

### 5.3 딜 등록

**경로:** 소싱관리 > 딜플로우관리 > 딜 등록

새로운 딜을 시스템에 등록합니다.

**등록 절차:**
1. 기업(고객)명 검색 (스타트업 관리에 먼저 등록 필요)
2. 검색 결과에서 기업 선택
3. 발굴일자, 섹터, 단계, 발굴경로 입력
4. 발굴자 선택
5. 한줄 소개 및 상세 설명 작성
6. "등록" 버튼 클릭

**자동화:** 등록 시 DealFlow "유입(inbound)" 단계 자동 생성

### 5.4 딜 상세 (BHV 스타일)

**경로:** 딜 목록에서 기업명 클릭

딜의 모든 정보를 한 화면에서 확인할 수 있습니다.

**표시 섹션 (7개):**
1. 기업 발굴정보: 기업명, 발굴일자, 섹터, 단계, 발굴경로, 발굴자, 상세설명
2. 기업 기본정보: 기업명, 설립일자, 법인번호, 사업자번호, 대표자, 임직원수, 소재지
3. 사업정보: 표준산업분류(KSIC), 주요사업, 상장시장, 상장일자
4. 재무정보: 자산총액, 자본금, 매출액, 영업이익
5. 연구개발: 기술연구소 유무, 연구인력
6. 연락처 정보: 도시, 웹사이트, 실무담당, 연락처, 이메일
7. 딜플로우 이력: 단계 이동 기록 (타임라인)

**주요 버튼:** 수정(편집 페이지 이동), 삭제(확인 대화상자)

### 5.5 딜 수정 (v1.5 신규)

**경로:** 딜 상세 페이지에서 "수정" 버튼 클릭 (`/sourcing/deals/[id]/edit`)

딜의 발굴정보를 BHV 스타일로 수정합니다.

**수정 가능 항목:**
- 기업(고객)명 (기업 변경 가능)
- 발굴일자, 섹터, 단계, 발굴경로, 발굴자
- 한줄 소개, 상세 설명

### 5.6 1차 스크리닝

**경로:** 소싱관리 > 스크리닝관리 > 1차 스크리닝 작성

**평가 항목 (SRC-F02):**

- 종합 점수 (자동 계산)
- 추천: 통과/보류/탈락

**자동화:** A등급 + 인계=Y → Handover 자동 생성 + 심사팀 알림

### 5.7 심사팀 인계

**인계 워크플로우:**
1. Sourcing팀: 인계 패키지 생성
2. 시스템: 심사팀에 알림 자동 발송
3. 심사팀: 인계 확인(Acknowledge) 클릭
4. 미확인 시: 매시간 자동 에스컬레이션

```
  스타트업 등록 → 딜 등록 → 칸반 "유입"
      ↓
[2단계: 1차 스크리닝] (Sourcing팀)
  SRC-F02 스크리닝 → 통과/보류/탈락
  A등급 + 인계=Y → 인계 패키지 자동 생성
      ↓
[3단계: 심사팀 인계] (Sourcing → 심사)
  인계 패키지 생성 → 심사팀 알림 → 확인(Acknowledge)
      ↓
[4단계: 심층 심사] (심사팀)
  서류심사 5축 → 인터뷰 → DD 13항목 → 투자메모
      ↓
[5단계: IC 상정/결정] (심사팀 → 파트너)
  IC 안건 등록 → 승인/조건부/보류/보육우선/부결
  승인 시 → Contract + Incubation 자동 생성
      ↓
[6단계: 투자 계약] (백오피스)
  텀시트 → 법률검토 → 서명 → 집행 → 17항목 클로징
  Cap Table 자동 업데이트
      ↓
[7단계: 보육 프로그램] (보육팀)
  온보딩 → 90일 액션플랜 → 멘토 매칭 → 월간 KPI 입력
  KPI 3개월 하락 → 위기 알림
      ↓
[8단계: 수요기업 연결] (OI팀)
  파트너 수요 파악 → 매칭 → PoC → 전환 결과 추적
  전환가능성 "높음" → 전략투자 검토 알림
      ↓
[9단계: 후속투자/EXIT]
  후속 라운드 유치 → EXIT → LP 보고서 생성
```

### 12.2 자동화 규칙 (10가지)

| # | 트리거 | 자동 액션 |
|---|--------|----------|
| 1 | 기업 등록 (SRC-F01) | Startup + DealFlow "유입" 자동 생성 |
| 2 | 스크리닝 A등급 + 인계=Y | Handover 자동 생성 + 심사팀 알림 |
| 3 | DD 13개 항목 전체 수령 | DD 완료 상태 자동 전환 |
| 4 | IC 승인 (INV-F03) | Contract + Incubation 동시 자동 생성 |
| 5 | 계약 17개 항목 완료 | Contract "클로징" + Cap Table 업데이트 |
| 6 | 멘토링 액션아이템 등록 | Task 자동 등록 + 기한 알림 |
| 7 | KPI 3개월 연속 하락 | 위기 알림 + 포트폴리오 등급 재검토 |
| 8 | PoC 전환가능성 "높음" | 심사팀에 전략투자 검토 알림 |
| 9 | 보고 마감 7/3/당일 | 자동 리마인더 발송 |
| 10 | 모든 양식 제출 | ActivityLog에 자동 기록 |

## 13. AI 에이전트 활용

eLSA는 MCP(Model Context Protocol) 서버를 통해 AI 에이전트와 연동됩니다.

### 13.1 사용 가능한 AI 에이전트

| 에이전트 | 기능 | 사용 시점 | 출력물 |
|---------|------|----------|--------|
| application-screener | 지원서 자동 스크리닝 | 지원서 접수 후 1차 평가 | 스크리닝 보고서 (100점 만점) |
| ir-analyst | IR 자료 심층 분석 | 심사 단계에서 IR 분석 | SWOT + 핵심질문 10개 |
| financial-analyst | 재무 건전성 평가 | DD 단계 재무 분석 | 재무 분석 보고서 |
| market-researcher | 시장/경쟁 분석 | 시장 조사 필요 시 | TAM/SAM/SOM + 경쟁 포지셔닝 |
| portfolio-reporter | 포트폴리오 현황 보고 | 정기 보고서 생성 | 포트폴리오 현황 보고서 |
| deal-review | 투자 심사 풀 사이클 | 스크리닝~투자메모 자동 | 스크리닝 + IR + 투자메모 + 멘토매칭 |
| mentor-match | 멘토 매칭 | 온보딩 시 적합 멘토 추천 | 멘토 매칭 추천서 |
| risk-alert | 리스크 경보 | 위기 신호 자동 감지 | 5축 리스크 매트릭스 |

### 13.2 AI 보고서 저장 위치

| 유형 | 경로 |
|------|------|
| 스크리닝 결과 | `ai-agent/reports/screening/` |
| 분석 보고서 | `ai-agent/reports/analysis/` |
| 투심위 자료 | `ai-agent/reports/investment-committee/` |
| LP 보고서 | `ai-agent/reports/lp/` |
| 리스크 경보 | `ai-agent/reports/risk/` |
| 운영 대시보드 | `ai-agent/reports/operations/` |

### 13.3 AI 활용 원칙

1. **투명성**: AI가 작성/분석한 결과물은 AI 활용 사실을 명시
2. **인간 감독**: AI 산출물은 반드시 담당자가 검토 후 최종 확정
3. **편향 방지**: 특정 산업/배경에 대한 편향이 없도록 모니터링
4. **책임**: AI 분석 결과에 기반한 의사결정의 책임은 담당자에게 있음

## 15. FAQ / 문제 해결

### Q1. 로그인이 안됩니다
- 이메일과 비밀번호를 정확히 입력했는지 확인
- 계정이 비활성화되었을 수 있으므로 관리자에게 문의
- 토큰 만료(24시간) 시 재로그인 필요

### Q2. 딜 수정이 안됩니다
- 딜 상세 페이지에서 "수정" 버튼 클릭 → 편집 페이지 이동
- 수정 권한이 있는 역할/팀에 소속되어야 합니다
- **주의:** API는 PATCH 메서드를 사용합니다 (PUT 아님)

### Q3. 칸반보드에서 드래그가 안됩니다
- 단계 이동은 유효한 전환 경로만 허용됩니다
- 예: 유입 → 1차 스크리닝 (가능), 유입 → IC심사 (불가)
- 권한이 없는 단계로의 이동은 차단됩니다

### Q4. 화면이 업데이트되지 않습니다
- 브라우저 강력 새로고침: Ctrl + Shift + R
- 캐시 삭제 후 다시 시도
- Docker 환경에서 프론트엔드 변경 미반영 시: `docker compose restart frontend`

### Q5. 에러 메시지가 표시됩니다

| 에러 코드 | 의미 | 해결 방법 |
|-----------|------|----------|
| 401 Unauthorized | 로그인 세션 만료 | 다시 로그인 |
| 403 Forbidden | 권한 없음 | 관리자에게 역할/팀 변경 요청 |
| 404 Not Found | 데이터 없음 | 해당 데이터가 삭제되었거나 존재하지 않음 |
| 405 Method Not Allowed | API 메서드 불일치 | 시스템 관리자에게 문의 |
| 422 Unprocessable | 입력값 오류 | 입력 내용 확인 (필수 필드, 형식) |

### Q6. 스타트업 등록 시 번호 검증 오류
- 법인등록번호: 13자리 숫자 (자동 하이픈 삽입)
- 사업자등록번호: 10자리 숫자 (자동 하이픈 삽입, 체크섬 검증)
- 형식이 맞지 않으면 등록이 차단됩니다

### Q7. LP 정보를 수정하고 싶습니다
- 백오피스 > LP 관리 > 해당 LP 클릭 > LP 상세 페이지
- "편집" 버튼 클릭 → LP 편집 페이지에서 수정

### Q8. AI 에이전트 보고서를 신뢰할 수 있나요?
- AI 보고서는 참고 자료입니다. 최종 의사결정은 반드시 담당자가 수행합니다
- AI 분석에는 "[추정]", "[미확인]" 표기가 포함될 수 있습니다
- 모든 AI 보고서 상단에 AI 활용 사실이 명시됩니다

### Q9. 위기 알림을 받았습니다
- KPI > 해당 기업 KPI 페이지에서 상세 확인
- 위기 유형: 현금고갈(런웨이 3개월 미만), 핵심인력 이탈, 고객 이탈 등
- 보육팀 PM과 즉시 대응 계획 수립 필요

### Q10. 접속 주소가 변경되었습니다
- 로컬: 항상 `http://localhost:3000`
- 외부: `https://winlsa.com` (Cloudflare Tunnel)
- CORS 오류 발생 시 관리자에게 도메인 등록 요청
