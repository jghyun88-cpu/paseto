# 대표 보고용 1페이지를 잘 쓰는 기준

이 양식은 상세함보다 **의사결정 속도**가 중요합니다.
따라서 아래 원칙으로 쓰는 것이 좋습니다.

첫째, 숫자는 많아도 **8개 내외 핵심지표**만 남깁니다.
둘째, 포트폴리오는 전체를 다 쓰지 말고 **상위 3~5개 핵심 기업만 기재**합니다.
셋째, 리스크는 전부 적지 말고 **대표 판단이 필요한 것만 3개 이내**로 제한합니다.
넷째, 보고서의 목적은 설명이 아니라 **다음 액션 합의**라는 점을 유지해야 합니다.

# 반도체·모빌리티 특화형으로 바꿀 때 추가할 항목

사용자님 조직처럼 산업특화 액셀러레이터라면 아래 4개를 대표 보고용에 추가하는 것이 좋습니다.

* **TRL 변화**
* **PoC 단계**
* **TIPS 준비도**
* **CVC/전략 파트너 적합성**

예를 들면 포트폴리오 표를 아래처럼 바꿀 수 있습니다.

| 기업명 | TRL | PoC 상태    | TIPS 준비도 | CVC 적합성  | 상태       |
| --- | --- | --------- | -------- | -------- | -------- |
|     |     | 미착수/진행/완료 | 높음/중간/낮음 | 높음/중간/낮음 | 정상/주의/위험 |

이렇게 바꾸면 대표가 딥테크 특유의 진행상황을 훨씬 빨리 읽을 수 있습니다.

# 가장 추천하는 실전 운영 방식

실제로는 보고서를 3단계로 나누면 가장 효율적입니다.

* **실무 상세표**: 팀 내부 관리용
* **팀장 보고표**: 중간 정리용
* **대표 보고 1페이지**: 최종 의사결정용

즉, 방금 만든 1페이지는 원본 데이터가 아니라
**실무표를 압축한 최종 대시보드**로 써야 합니다.
