# 보고서 품질 벤치마크 기준

> 에이전트 출력물의 품질을 측정하기 위한 벤치마크 정의

---

## 1. 등급 체계

| 등급 | 점수 범위 | 의미 | 후속 조치 |
|------|----------|------|----------|
| A (우수) | 90-100% | 즉시 사용 가능 | 없음 |
| B (양호) | 75-89% | 경미한 수정 후 사용 | 빠른 검토 |
| C (보통) | 60-74% | 수정 필요 | 상세 검토 + 수정 |
| D (미흡) | 40-59% | 대폭 수정 필요 | 재작성 검토 |
| F (부적합) | 0-39% | 사용 불가 | 재작성 |

## 2. 공통 품질 항목 (10점)

| # | 항목 | 배점 | 기준 |
|---|------|------|------|
| 1 | 메타 정보 | 1 | YAML frontmatter 또는 상단 메타 블록 존재 |
| 2 | 작성일 | 1 | YYYY-MM-DD 형식 날짜 포함 |
| 3 | 기밀등급 | 1 | 내부용/기밀/대외비/공개 중 하나 명시 |
| 4 | 정량적 근거 | 1 | 수치+단위 패턴 3건 이상 |
| 5 | 불확실성 표기 | 1 | 추정 정보에 [미확인]/[추정] 표기 |
| 6 | 균형성 | 1 | 긍정/부정 양면 분석 존재 |
| 7 | 구조화 테이블 | 1 | 마크다운 테이블 1개 이상 |
| 8 | 적정 분량 | 1 | 500자~50,000자 범위 |
| 9 | 섹션 구조 | 1 | H2/H3 헤딩 3개 이상 |
| 10 | 용어 영문 병기 | 1 | 전문 용어에 영문 병기 1건 이상 |

## 3. 유형별 추가 항목 (각 4점)

## 4. 목표 KPI

| 지표 | 목표치 | 측정 방법 | 점검 주기 |
|------|--------|----------|----------|
| 수정 필요 비율 | < 20% | B등급 미만 비율 | 주간 |
| 부적합 비율 | < 5% | F등급 비율 | 주간 |
| 평균 품질 점수 | > 75% | 전체 보고서 평균 | 월간 |
| A등급 비율 | > 50% | A등급 보고서 비율 | 월간 |
| 유용성 점수 | > 4/5 | 담당자 만족도 설문 | 분기 |
| 시간 절감율 | > 60% | AI vs 수동 시간 비교 | 분기 |

## 5. 품질 개선 프로세스

```
[1] 자동 점검: quality-benchmark.mjs batch
       │
[2] 결과 분석: 미충족 항목 빈도 파악
       │
[3] 원인 진단:
       ├── 공통 항목 미충족 → 에이전트 공통 지침 개선
       ├── 유형별 항목 미충족 → 해당 에이전트 프롬프트 보강
       └── 데이터 부족 → data/reference/ 보강
       │
[4] 개선 적용: 규칙/에이전트/데이터 업데이트
       │
[5] 재측정: 다음 batch 점검에서 개선 확인
```

---

> 이 벤치마크는 `node scripts/quality-benchmark.mjs` 로 자동 측정되며,
> 결과는 `logs/quality-benchmark.log` 에 누적 기록됩니다.
